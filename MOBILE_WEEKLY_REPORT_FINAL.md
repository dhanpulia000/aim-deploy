# Mobile 주간보고서 자동 생성 가이드

## 주간보고서 시트 구성 (최종)

1. **[10월5주차]** - 메인 시트
2. **주요 이슈 건수 증감** - 전주 대비 증감 분석
3. **공유 이슈 시간 순** - 시간순 공유된 중요 이슈
4. **VoC** - 일일보고서 VoC 데이터 집계
5. **Data** - 주간 요약 데이터 (WEEKLY SUMMARY DATA)

## 시트별 데이터 매핑

### 1. 메인 시트 ([10월5주차])

#### 차트 1: 성향별 주간 동향 수
- **데이터 소스**: VoC 시트의 `성향` 컬럼
- **집계**: 긍정/부정/중립 건수

#### 차트 2: 이슈별 주간 동향 수
- **데이터 소스**: VoC `대분류` + Issue `분류`
- **카테고리**: 게임 플레이 문의, 버그, 서버/접속, 불법프로그램, 비매너/이스포츠

#### 부정/긍정/기타 동향 요약
- **데이터 소스**: VoC + Issue 시트
- **대분류별 상세 내용 추출**

#### 모니터링 업무 현황
- **데이터 소스**: Summary 시트의 `취합량`
- **일별/주별 집계**

### 從而. 주요 이슈 건수 증мат

- **9개 주요 카테고리** 전주 대비 증감 분석
- **MO 취합량** 대비 비율 계산
- **증감률 및 방향** (▲/▼) 표시

### 3. 공유 이슈 시간 순

- **Severity 1-2 이슈** 필터링
- **시간순 정렬**
- **날짜 + 상태** 표시

### 4. VoC 시트

- **일일보고서 VoC 데이터** 집계
- **기간 필터링** 후 날짜순 정렬
- **일일보고서 VoC와 동일한 구조**

### 5. Data 시트 (WEEKLY SUMMARY DATA)

#### 컬럼 구조
| 컬럼명 | 설명 | 데이터 소스 |
|--------|------|------------|
| 주차 | 주차 정보 | 일일보고서 Data |
| 날짜 | YYYY-MM-DD | 일일보고서 Data |
| 컨텐츠 | 공지사항, 패치 내용 | 요청/비고에서 추출 |
| 커뮤니티 이슈 | 시간과 공유자 포함 | Issue/VoC 중요 이슈 |
| 이용자 동향 | 이용자 동향 요약 | VoC 데이터 요약 |
| 공유 | 공유 요청 사항 | 메인 시트 "요청 사항" |
| 요청 | 요청 사항 | 메인 시트 "요청 사항" |
| 비고 | 패치 시간, 버그 수정 | 메인 시트 "비고" |

## 일일보고서 → 주간보고서 변환 흐름

```
일일보고서 5개 시트 업로드
├─ Summary → 모니터링 업무 현황 (취합량)
├─ Data → 주차 정보 (Data 시트용)
├─ Issue → 버그/서버 이슈
├─ Cheat&Abuse → 불법 프로그램 제보
└─ VoC → 이용자 동향
         ↓
기간 필터링 (예: 10/27 ~ 11/02)
         ↓
데이터 집계 및 변환
├─ 성향별/이슈별 집계
├─ 부정/긍정/기타 동향 정리
├─ 주요 이슈 건수 증감 계산
├─ 공유 이슈 추출
└─ 모니터링 현황 통계
         ↓
주간보고서 생성
```

## 구현 완료 상태

✅ 주간보고서 구조 분석 완료
✅ 일일보고서 → 주간보고서 매핑 완료
⏳ `generateMobileWeeklyReport` 함수 구현 필요
