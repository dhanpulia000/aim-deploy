# 에이전트 계정 비밀번호 재설정 가이드

## 현재 상황

현재 시스템에는 UI를 통한 비밀번호 재설정 기능이 구현되어 있지 않습니다. 비밀번호를 잊은 경우 다음 방법을 사용할 수 있습니다.

## 방법 1: 스크립트를 사용한 비밀번호 재설정 (권장)

### 사용법

```bash
cd backend
node scripts/reset-password.js <이메일> <새비밀번호>
```

### 예시

```bash
# 에이전트 계정 비밀번호 변경
node scripts/reset-password.js agent@example.com newpassword123
```

### 실행 결과

```
✅ 비밀번호가 성공적으로 변경되었습니다!

사용자 정보:
  이메일: agent@example.com
  이름: Agent Name
  역할: AGENT
  연결된 에이전트: Agent Name

새 비밀번호: newpassword123

⚠️  이 비밀번호를 안전하게 보관하세요.
```

## 방법 2: 데이터베이스에서 직접 확인 및 변경

### 1. 사용자 계정 확인

```bash
cd backend
node scripts/checkLoginableAgents.js
```

이 스크립트는 모든 로그인 가능한 계정을 보여줍니다.

### 2. 비밀번호 재설정

방법 1의 스크립트를 사용하거나, 직접 데이터베이스를 수정할 수 있습니다.

## 방법 3: 관리자 페이지에서 에이전트 재생성 (비권장)

1. 관리자 페이지 → 에이전트 관리
2. 기존 에이전트 삭제
3. 동일한 이메일로 새 에이전트 생성 (새 비밀번호 설정)

⚠️ **주의**: 이 방법은 에이전트의 모든 데이터를 삭제하므로 권장하지 않습니다.

## 보안 권장사항

1. **비밀번호 정책**:
   - 최소 6자 이상
   - 영문, 숫자, 특수문자 조합 권장
   - 정기적인 비밀번호 변경

2. **접근 제어**:
   - 비밀번호 재설정 스크립트는 관리자만 실행 가능하도록 제한
   - 재설정 후 비밀번호를 안전하게 전달

3. **감사 로그**:
   - 비밀번호 변경 시 감사 로그 기록 (향후 구현 예정)

## 향후 개선 사항

다음 기능들이 추가될 예정입니다:

1. **사용자 비밀번호 변경 기능**:
   - 로그인한 사용자가 자신의 비밀번호 변경
   - 현재 비밀번호 확인 후 변경

2. **관리자 비밀번호 재설정 기능**:
   - 관리자 페이지에서 다른 사용자의 비밀번호 재설정
   - 권한 확인 및 감사 로그

3. **비밀번호 재설정 이메일**:
   - 이메일을 통한 비밀번호 재설정 링크 발송
   - 임시 토큰 기반 재설정

## 문제 해결

### 스크립트 실행 시 오류

**오류: "사용자를 찾을 수 없습니다"**
- 이메일 주소가 정확한지 확인
- `checkLoginableAgents.js`로 사용자 목록 확인

**오류: "비밀번호는 최소 6자 이상이어야 합니다"**
- 새 비밀번호를 6자 이상으로 설정

**오류: 데이터베이스 연결 실패**
- `.env` 파일의 `DATABASE_URL` 확인
- 데이터베이스 파일이 존재하는지 확인

## 관련 파일

- `backend/scripts/reset-password.js`: 비밀번호 재설정 스크립트
- `backend/scripts/checkLoginableAgents.js`: 로그인 가능한 계정 확인 스크립트
- `backend/services/auth.service.js`: 인증 서비스
- `backend/controllers/auth.controller.js`: 인증 컨트롤러
