# Mobile 주간보고서 테스트 가이드

## 테스트 시나리오

### 1단계: 백엔드 서버 실행 확인

```bash
# 백엔드가 실행 중인지 확인
# http://localhost:8080 에 접속 가능해야 합니다
```

### 2단계: 일일보고서 업로드

1. 브라우저에서 `http://localhost:5173/login` 접속
2. 에이전트 ID 입력 (예: `admin`)
3. `/dashboard` 페이지로 이동
4. **파일 타입 선택**: "Mobile 모니터링 일일 보고서.xlsx"
5. **파일 업로드**: `backend/data/20251027 PUBG MOBILE 모니터링 일일 보고서.xlsx`
6. 여러 일일보고서를 업로드 (최소 3-5개 권장)

### 3단계: 주간보고서 생성

#### 방법 1: UI를 통한 생성
1. 브라우저에서 `http://localhost:5173/weekly-report` 접속
2. 시작 일시와 종료 일시 입력
3. "Mobile 보고서 생성" 버튼 클릭
4. 결과 확인

#### 방법 2: 지난 주 자동 생성
1. `/weekly-report` 페이지에서
2. "Mobile 주간 보고서" 섹션의 "지난 주 자동 생성" 버튼 클릭
3. 결과 확인

### 4단계: 결과 확인 항목

✅ **성향별 주간 동향**
- 긍정/부정/중립 건수 표시

✅ **이슈별 주간 동향**
- 게임 플레이 문의, 버그, 서버/접속 등 표시

✅ **주요 이슈 건수**
- 9개 카테고리별 건수 표시

✅ **공유 이슈**
- Severity 1-2 이슈 시간순 정리

✅ **부정/긍정/기타 동향 요약**
- 각 동향별 상세 내용

✅ **VoC 전체 데이터**
- 날짜순 정렬된 이용자 동향

## 문제 해결

### 문제: "보고서 생성 실패"
**원인**: 일일보고서 데이터가 없음
**해결**: 먼저 일일보고서를 업로드하세요

### 문제: 결과가 비어 있음
**원인**: 선택한 기간에 일일보고서가 없음
**해결**: 다른 기간을 선택하거나 일일보고서를 추가로 업로드하세요

### 문제: 데이터가 잘못 집계됨
**원인**: 파싱 로직 오류
**해결**: 백엔드 콘솔 로그를 확인하세요
