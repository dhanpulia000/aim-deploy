1. 시작하기

1.1 시스템이란?

Agent Ops Wallboard V2는 게임 운영팀을 위한 실시간 이슈 모니터링 및 관리 시스템입니다.

주요 기능:

  • 네이버 카페, 슬랙에서 발생하는 이슈를 자동으로 수집
  • AI가 이슈를 자동으로 분류하고 우선순위를 설정
  • 이슈 처리 상태를 실시간으로 추적
  • AI 어시스턴트: RAG 기반 업무 가이드 검색 및 질의응답
  • 데스크톱 알림: 새 이슈 및 중요 알림을 데스크톱에 표시
  • 공지사항 관리: 날짜/주차/담당자/게임별 필터링 및 열람 확인
  • 업무 체크리스트: 날짜·작업 구분·요일별로 실행 여부 체크
  • 일일/주간 보고서 자동 생성
  • 채널별 이슈 보기: 네이버 카페 / Discord / 시스템 등 수집 소스별로 메인 보드를 나누어 조회
  • 포럼 모니터링(inZOI 공식 포럼·Discourse): 티켓과 별도의 일일 동향 리포트 화면

1.2 로그인하기

1단계: 시스템 접속

  1. 관리자에게 받은 시스템 URL을 브라우저 주소창에 입력합니다
    • 예: http://localhost:8080 또는 https://wallboard.example.com
  2. 로그인 페이지가 표시됩니다

2단계: 계정 정보 입력

  1. 이메일 주소를 입력합니다
    • 예: agent@example.com
  2. 비밀번호를 입력합니다
  3. "로그인" 버튼을 클릭합니다

💡 팁: 비밀번호를 잊어버렸다면 관리자에게 문의하세요.

3단계: 로그인 확인

  • 로그인이 성공하면 메인 화면(이슈 큐)으로 이동합니다
  • 우측 상단에 본인의 이름이 표시되면 정상적으로 로그인된 것입니다

1.3 화면 구성 이해하기

2. 메인 화면 이해하기

2.1 프로젝트 선택

위치: 화면 상단 왼쪽

사용 방법:

  1. 드롭다운 메뉴를 클릭합니다
  2. 작업할 프로젝트를 선택합니다
    • "전체 프로젝트": 모든 프로젝트의 이슈를 표시
    • "PUBG PC": PUBG PC 프로젝트 이슈만 표시
    • "PUBG Mobile": PUBG Mobile 프로젝트 이슈만 표시
  3. 선택한 프로젝트의 이슈만 화면에 표시됩니다

💡 팁: 프로젝트를 선택하면 해당 프로젝트의 에이전트 정보도 함께 표시됩니다.

2.2 KPI 대시보드

위치: 화면 상단 중앙

표시되는 정보:

  • 열린 이슈: 현재 처리 대기 중인 이슈 수
  • Sev1 이슈: 심각도 1(긴급) 이슈 수
  • SLA 임박: SLA 마감 시간이 임박한 이슈 수
  • 평균 처리 시간: 이슈 처리 평균 시간

⚠️ 주의: SLA 임박 이슈는 즉시 처리해야 합니다!

2.3 중요 이슈 섹션

위치: 이슈 큐 상단

표시되는 이슈:

  • 심각도 1(Sev1) 이슈
  • SLA 마감 시간이 임박한 이슈
  • 최대 20개까지 표시 (스크롤하면 추가 로딩)

특징:

  • 빨간색 배경으로 강조 표시
  • 우선적으로 처리해야 하는 이슈

2.4 일반 이슈 큐

위치: 중요 이슈 섹션 아래

표시되는 이슈:

  • 심각도 2, 3 이슈
  • 처리 대기 중인 모든 이슈
  • 최대 1000개까지 표시

뷰 모드:

  • 카드 뷰: 이슈를 카드 형식으로 표시 (기본)
  • 리스트 뷰: 이슈를 테이블 형식으로 표시

전환 방법:

  1. 이슈 큐 상단의 "카드 뷰" 또는 "리스트 뷰" 버튼을 클릭합니다

2.5 에이전트 상태 패널

위치: 화면 우측

표시되는 정보:

  • 로그인 중인 에이전트: 파란색 배경으로 강조 표시
  • 기타 에이전트: 일반 배경

각 에이전트 정보:

  • 이름과 상태 (🟢 온라인, 🟡 자리비움, 🔴 오프라인)
  • 담당 게임 목록
  • 오늘 처리한 이슈 수
  • 현재 처리 중인 이슈 수
  • 평균 처리 시간

💡 팁: 본인의 정보는 "로그인 중" 섹션에 표시됩니다.

2.6 검색 기능

위치: 화면 상단 중앙

사용 방법:

  1. 검색 필드에 키워드를 입력합니다
  2. 검색 결과가 실시간으로 필터링됩니다

검색 대상:

  • 이슈 제목
  • 이슈 내용
  • 이슈 ID
  • 카테고리 이름
  • 담당 에이전트 이름

검색 예시:

  • 버그 → "버그"가 포함된 이슈 검색
  • 접속 불가 → "접속 불가"가 포함된 이슈 검색
  • iron.lee → iron.lee가 담당자인 이슈 검색

💡 팁: 검색어를 지우면 전체 목록이 다시 표시됩니다.

2.7 수동 수집 기능

위치: 화면 상단 오른쪽, "➕ 수동 수집" 버튼

언제 사용하나요?

  • 크롤러가 놓친 게시글을 즉시 수집해야 할 때
  • 긴급하게 수집해야 하는 게시글이 있을 때

사용 방법:

  1. "➕ 수동 수집" 버튼을 클릭합니다
  2. 모달 창이 열립니다
  3. 네이버 카페 게시글 URL을 입력합니다
    • 예: https://cafe.naver.com/example/12345
    • 모바일 URL (m.cafe.naver.com)도 자동으로 PC 버전으로 변환됩니다
  4. "수집 시작" 버튼을 클릭합니다
  5. 수집이 완료되면 "수집 완료" 메시지가 표시됩니다
  6. 이슈 목록이 자동으로 갱신됩니다

⚠️ 주의:

  • URL은 반드시 cafe.naver.com 도메인이어야 합니다
  • 비공개 게시글은 수집할 수 없습니다
  • 수집에는 약 10-30초가 소요될 수 있습니다

2.8 모니터링(수집 소스) 탭

위치: 메인 화면(이슈 보드)에서 업무 시작 전 바로가기 카드 위

역할: 티켓 이슈를 어느 채널(플랫폼)에서 수집했는지에 따라 목록을 바로 필터링합니다.

탭 구성:

  • 이슈 · 전체: 네이버, Discord, 시스템 등 모든 소스의 티켓
  • 이슈 · 네이버 카페: 네이버 카페 크롤러로 들어온 티켓만
  • 이슈 · Discord: Discord로 들어온 티켓만
  • 이슈 · 시스템·기타: 시스템·기타 소스 티켓만

💡 팁: 선택 상태는 브라우저에 저장되어 새로고침 후에도 유지되는 경우가 많습니다.
💡 주의: 「수집 구분」(검색·필터 안의 드롭다운)은 게임/크롤러 코드(어느 카페·보드인지)를 고르는 필터입니다. 위 탭의 플랫폼(네이버·Discord)과는 다른 차원입니다.

3. 이슈 처리하기

3.1 이슈 카드 이해하기

이슈 카드에는 다음 정보가 표시됩니다:

┌─────────────────────────────────────────┐
│ [Sev1] 🔴 접속 불가 문제                │
│ 네이버 | 2025-12-20 14:30              │
│ 카테고리: 장애/접속                     │
│ 담당자: iron.lee                        │
│ 상태: 열림                              │
│ [원문 링크] [스크린샷]                  │
└─────────────────────────────────────────┘

각 요소 설명:

  • 심각도 배지: [Sev1], [Sev2], [Sev3]
  • 제목: 이슈 제목
  • 소스: 네이버, 슬랙 등
  • 시간: 원본 게시글 작성 시간
  • 카테고리: AI가 자동으로 분류한 카테고리
  • 담당자: 담당 에이전트 이름
  • 상태: 열림, 진행중, 완료 등
  • 원문 링크: 원본 게시글로 이동하는 링크
  • 스크린샷: 게시글 스크린샷 이미지 (있는 경우)

3.2 이슈 상세 보기

열기:

  1. 이슈 카드를 클릭합니다
  2. 우측에 상세 패널이 열립니다

상세 패널 구성:

3.2.1 기본 정보

  • 제목: 이슈 제목
  • 소스: 네이버 카페, 슬랙 등
  • 심각도: Sev1, Sev2, Sev3
  • 카테고리: 대분류/중분류
  • 작성 시간: 원본 게시글 작성 시간
  • 원문 링크: 원본 게시글로 이동

3.2.2 AI 분류 결과

  • 보라색 박스: AI 기반 분류 결과
  • 파란색 박스: 규칙 기반 분류 결과
  • 분류 이유: AI가 이슈를 해당 카테고리로 분류한 이유

3.2.3 이슈 내용

  • 게시글 본문 내용
  • 스크린샷 이미지 (있는 경우)
  • 댓글 정보 (수집된 경우)

3.3 이슈 상태 변경하기

상태란? 이슈의 현재 처리 단계를 나타냅니다.

상태 종류:

  • OPEN (열림): 새로 수집된 이슈, 아직 처리하지 않음
  • TRIAGED (분류 완료): 이슈를 분류하고 우선순위를 정함
  • IN_PROGRESS (진행 중): 현재 처리 중인 이슈
  • WAITING (대기 중): 다른 팀이나 담당자의 응답을 기다리는 중
  • RESOLVED (완료): 처리 완료된 이슈
  • VERIFIED (검증 완료): 완료 후 검증까지 완료된 이슈

상태 변경 방법:

  1. 이슈 상세 패널을 엽니다
  2. "상태" 드롭다운을 클릭합니다
  3. 원하는 상태를 선택합니다
  4. 변경 사항이 자동으로 저장됩니다

일반적인 워크플로우:

OPEN → TRIAGED → IN_PROGRESS → RESOLVED

💡 팁: 상태를 변경하면 이슈 목록에서도 즉시 반영됩니다.

3.4 담당 에이전트 할당하기

할당이란? 이슈를 처리할 에이전트를 지정하는 것입니다.

할당 방법:

  1. 이슈 상세 패널을 엽니다
  2. "담당자 할당" 드롭다운을 클릭합니다
  3. 에이전트 목록에서 담당자를 선택합니다
  4. 선택한 에이전트가 이슈에 할당됩니다

할당 확인:

  • 이슈 카드에 담당자 이름이 표시됩니다
  • 에이전트 상태 패널에서 해당 에이전트의 "진행중" 수가 증가합니다

💡 팁: 본인에게 할당된 이슈는 우선적으로 처리하세요.

3.5 댓글 작성하기

댓글이란? 이슈에 대한 메모나 처리 내역을 기록하는 것입니다.

댓글 작성 방법:

  1. 이슈 상세 패널을 엽니다
  2. 하단의 **"댓글 입력 필드"**에 내용을 입력합니다
  3. "댓글 추가" 버튼을 클릭합니다
  4. 댓글이 추가되고 목록에 표시됩니다

댓글 활용 예시:

  • 처리 내역 기록: "원인 확인 완료, 수정 진행 중"
  • 메모: "관련 이슈 #12345 참고"
  • 협업: "개발팀에 문의 필요"

댓글 확인:

  • 이슈 상세 패널 하단의 "댓글" 섹션에서 확인할 수 있습니다
  • 작성자, 작성 시간이 함께 표시됩니다

3.6 여러 이슈 일괄 처리하기

일괄 처리란? 여러 이슈를 한 번에 처리하는 기능입니다.

일괄 처리 방법:

  1. 이슈 목록에서 체크박스를 클릭하여 여러 이슈를 선택합니다
  2. 상단의 버튼 중 하나를 클릭합니다:
    • "선택 미열람": 선택한 이슈를 미열람 상태로 되돌림
    • "선택 완료": 선택한 이슈를 완료 처리
    • "선택 보고서 제외": 선택한 이슈를 보고서에서 제외하고 완료 처리

일괄 처리 예시:

  • 중복 이슈 10개를 선택 → "선택 보고서 제외" 클릭
  • 처리 완료된 이슈 5개를 선택 → "선택 완료" 클릭

⚠️ 주의: "선택 보고서 제외"는 되돌릴 수 없으므로 신중하게 사용하세요.

3.7 스크린샷 보기

스크린샷이란? 네이버 카페 게시글의 스크린샷 이미지입니다.

스크린샷 보기:

  1. 이슈 카드에 스크린샷 아이콘이 있으면 스크린샷이 있는 것입니다
  2. 이슈 상세 패널에서 스크린샷 이미지를 확인할 수 있습니다
  3. 이미지를 클릭하면 새 창에서 크게 볼 수 있습니다

💡 팁: 스크린샷은 게시글 수집 시 자동으로 생성됩니다.

4. 일일 업무 흐름

4.1 업무 시작하기

아침 업무 체크리스트:

  1. ✅ 시스템에 로그인
  2. ✅ 프로젝트 선택 (본인이 담당하는 프로젝트)
  3. ✅ KPI 대시보드 확인
    • 열린 이슈 수 확인
    • Sev1 이슈 수 확인
    • SLA 임박 이슈 확인
  4. ✅ 중요 이슈 섹션 확인
    • 우선 처리해야 할 이슈 확인

4.2 이슈 처리 순서

우선순위:

  1. Sev1 이슈 (심각도 1, 긴급)
  2. SLA 임박 이슈 (마감 시간이 임박한 이슈)
  3. 본인에게 할당된 이슈
  4. 일반 이슈 (Sev2, Sev3)

처리 단계:

  1. 이슈 카드를 클릭하여 상세 내용 확인
  2. 원문 링크를 클릭하여 원본 게시글 확인
  3. 필요시 스크린샷 확인
  4. 담당자 할당 (본인 또는 적절한 담당자)
  5. 상태를 "진행 중"으로 변경
  6. 처리 진행
  7. 댓글로 처리 내역 기록
  8. 완료 시 상태를 "완료"로 변경

4.3 업무 중 주의사항

실시간 업데이트:

  • 새로운 이슈가 자동으로 추가됩니다
  • 다른 에이전트가 처리한 이슈는 실시간으로 상태가 업데이트됩니다
  • 페이지를 새로고침할 필요가 없습니다

SLA 관리:

  • SLA 마감 시간이 임박한 이슈는 즉시 처리해야 합니다
  • KPI 대시보드의 "SLA 임박" 수를 주기적으로 확인하세요

협업:

  • 다른 에이전트에게 이슈를 할당할 때는 댓글로 이유를 남기세요
  • 처리 중인 이슈는 댓글로 진행 상황을 공유하세요

4.4 업무 종료 전 체크리스트

저녁 업무 체크리스트:

  1. ✅ 본인에게 할당된 이슈 처리 완료 확인
  2. ✅ 진행 중인 이슈 상태 업데이트
  3. ✅ 미처리 이슈 확인 및 다음 담당자에게 할당
  4. ✅ 댓글로 처리 내역 기록

5. 고급 기능

5.1 필터링 기능

필터란? 이슈 목록을 조건에 맞게 필터링하는 기능입니다.

필터 종류:

  • 수집 소스(플랫폼): 메인 보드 상단 모니터링 탭에서 선택합니다(네이버 / Discord / 시스템 / 전체). 검색·필터 패널 안의 예전 플랫폼 드롭다운은 탭으로 대체되었습니다.
  • 수집 구분(게임/크롤러): 검색·필터 영역의 드롭다운 — 어느 모니터링 보드·게임 코드에서 왔는지로 좁힙니다.
  • 심각도 필터: Sev1, Sev2, Sev3
  • 상태 필터: 열림, 진행중, 완료 등
  • 카테고리 필터: 버그, 문의, 건의 등

필터 사용 방법:

  1. 모니터링 탭 또는 검색 및 필터 영역을 펼칩니다
  2. 원하는 조건을 선택합니다
  3. 필터링된 결과가 표시됩니다

필터 조합:

  • 여러 필터를 동시에 사용할 수 있습니다
  • 예: 모니터링 탭 네이버 + 심각도 Sev1 + 상태 열림 → 네이버에서 온 Sev1 열림 이슈만 표시

5.2 뷰 모드 전환

카드 뷰:

  • 이슈를 카드 형식으로 표시
  • 한눈에 많은 정보를 확인할 수 있음
  • 기본 뷰 모드

리스트 뷰:

  • 이슈를 테이블 형식으로 표시
  • 많은 이슈를 빠르게 스캔할 수 있음
  • 상태, 시간, 대분류, 제목, 담당자 정보를 한 줄에 표시

전환 방법:

  • 이슈 큐 상단의 "카드 뷰" 또는 "리스트 뷰" 버튼을 클릭합니다

5.3 AI 어시스턴트

AI 어시스턴트란? 업무 가이드 문서를 기반으로 질문에 답변해주는 AI 챗봇입니다.

접근 방법:

  1. 메인 화면 상단의 "🤖 AI 어시스턴트" 버튼을 클릭합니다
  2. AI 어시스턴트 페이지로 이동합니다

사용 방법:

  1. 하단의 입력 필드에 질문을 입력합니다
    • 예: "일일 리포트 작성 방법은?"
    • 예: "이슈 처리 절차는?"
  2. Enter 키를 누르거나 전송 버튼을 클릭합니다
  3. AI가 업무 가이드 문서를 검색하여 답변을 제공합니다
  4. 관련 가이드 문서가 있으면 함께 표시됩니다

특징:

  • 업로드된 PDF 가이드 문서를 기반으로 답변
  • 의미 기반 검색으로 관련 문서 자동 찾기
  • 대화형 인터페이스로 자연스러운 질의응답

💡 팁: 질문을 구체적으로 작성할수록 더 정확한 답변을 받을 수 있습니다.

5.4 공지사항 페이지

공지사항 페이지란? 모든 공지사항을 한 곳에서 관리하고 확인할 수 있는 전용 페이지입니다.

접근 방법:

  1. 메인 화면 상단의 "📢 공지사항" 버튼을 클릭합니다
  2. 공지사항 페이지로 이동합니다

필터링 기능:

  • 날짜별: 특정 날짜의 공지사항만 표시
  • 주차별: 특정 주차의 공지사항만 표시
  • 담당자별: 특정 담당자의 공지사항만 표시
  • 게임별: 특정 게임의 공지사항만 표시
  • 검색: 제목이나 내용으로 검색

열람 확인:

  • 각 공지사항에 열람한 에이전트 목록이 표시됩니다
  • "확인" 버튼을 클릭하면 본인이 열람했다고 표시됩니다
  • 열람하지 않은 에이전트 목록도 확인할 수 있습니다

공지 작성 (LEAD, ADMIN 권한 필요):

  1. 메인 화면에서 공지 작성 버튼을 클릭합니다
  2. 다음 정보를 입력합니다:
    • 게임명
    • 담당자 이름
    • 카테고리
    • 공지 내용
    • 공지 날짜
  3. "저장" 버튼을 클릭합니다

5.5 데스크톱 알림 설정

데스크톱 알림이란? 새 이슈나 중요 알림을 브라우저 밖 데스크톱에 표시하는 기능입니다.

접근 방법:

  1. 메인 화면 상단의 "🔔 알림 설정" 버튼을 클릭합니다
  2. 알림 설정 페이지로 이동합니다

권한 요청:

  1. "알림 권한 요청" 버튼을 클릭합니다
  2. 브라우저에서 알림 권한을 허용합니다
  3. 권한이 허용되면 "✅ 허용됨"으로 표시됩니다

알림 설정:

  • 데스크톱 알림 활성화: 전체 알림 기능 켜기/끄기
  • 알림 소리: 알림 표시 시 소리 재생 여부
  • 새 이슈 알림: 새 이슈 생성 시 알림 표시
  • SLA 위반 알림: SLA 위반 시 알림 표시
  • 이슈 업데이트 알림: 이슈 상태 변경 시 알림 표시

알림 클릭:

  • 알림을 클릭하면 해당 이슈로 자동 이동합니다
  • 메인 화면으로 전환되고 이슈 상세 패널이 열립니다

💡 팁: 브라우저를 닫아도 알림은 계속 표시됩니다 (브라우저가 백그라운드에서 실행 중인 경우).

5.6 고객 피드백 공지 (메인 화면)

  2. 공지를 클릭하면 상세 내용을 확인할 수 있습니다

5.7 클랜 게시글 관리

위치

  • 상단 메뉴 → 관리 → 데이터 관리 → 클랜 게시글 관리

목적

  • PUBG PC / PUBG Mobile 카페의 클랜 모집·홍보 게시글만 따로 모아서 관리하는 화면입니다.
  • 일반 이슈 큐에서는 클랜 모집글을 최대한 숨기고, 이 화면에서만 집중해서 확인합니다.

화면 구성

  • 상단: 프로젝트 선택, 기간 필터, "알림 게시글만 보기" 토글 버튼
  • 표 컬럼: 알림 / 날짜 / 제목 / 게시판 / 작업
    • 날짜: 원본 게시글 작성 시각 (날짜 + 시간, 한국 시간 기준)
    • 제목: 카페에 노출되는 제목 그대로 표시
    • 게시판: 클랜/방송/디스코드, 클랜 홍보 등 게시판 이름
    • 작업:
      • "원문 보기": 새 탭에서 네이버 카페 원본 게시글 열기
      • "이슈로 이동" (있는 경우): 이미 이슈로 승격된 경우 이슈 상세로 바로 이동

알림(Alerts)

  • 제목 왼쪽의 작은 컬러 배지가 알림 유형을 의미합니다.
    • 예: 이모지 과다, 특수문자 이모티콘, 성별 표현, 중복 홍보 등
  • 알림별로 "해제" 버튼이 있으며, 누르면:
    • 해당 알림은 이 글에서만 숨겨집니다.
    • 상단/좌측 메뉴에 표시되는 클랜 알림 카운트도 즉시 줄어듭니다.
  • 한 글에 여러 알림이 있을 때는, "모두 해제" 버튼으로 한 번에 지울 수 있습니다.

"알림 게시글만 보기" 버튼

  • 버튼을 누르면:
    • 알림이 하나 이상 붙은 게시글만 필터링해서 보여줍니다.
    • 이때는 서버에서 더 넉넉하게 데이터를 가져온 뒤, 화면에서 추가 필터링합니다.
  • 다시 누르면 전체 클랜 게시글 목록으로 돌아갑니다.

💡 에이전트 가이드

  • 일반 VOC / 장애 이슈는 기존 이슈 큐에서 처리하고,
    클랜 모집글 관련 모니터링은 이 화면만 보면 됩니다.
  • 알림을 모두 해제해도 원본 글 자체는 삭제되지 않습니다.
    “문제 없음”으로 판단했다는 의미로 이해하면 됩니다.

5.8 파트너 영상 아카이빙

위치

  • 상단 메뉴 → 파트너 영상 아카이빙

기능 개요

  • 파트너 유튜브 채널들의 일주일치 영상 데이터를 자동으로 수집해서,
    • 조회수 / 좋아요 / 댓글 수 / 영상 유형을 CSV로 내려받는 도구입니다.
  • 엑셀에 정리된 채널 리스트를 업로드하면,
    • 각 채널의 주간 업로드 영상 목록 + 통계를 한 번에 뽑아줍니다.

엑셀 채널 리스트 형식

  • 기본 컬럼 예시:
    • channelName (채널 이름)
    • youtubeUrl (채널 URL)
    • channelId (선택, 가능하면 UC로 시작하는 공식 채널 ID 입력 권장)
  • 권장 방식:
    • 가능하면 channelId 컬럼에 UC로 시작하는 채널 ID를 직접 입력합니다.
    • 이렇게 하면 API 할당량을 거의 쓰지 않고, 검색 오류도 줄어듭니다.

사용 방법

  1. 페이지 상단에서 엑셀 파일 업로드 버튼을 눌러 채널 리스트를 선택합니다.
  2. 기준 주차(날짜)를 확인합니다.
  3. "아카이빙 시작" 버튼을 클릭합니다.
  4. 진행 상황이 화면에 표시되며, 채널별:
    • 성공한 채널 수 / 오류 채널 수
    • 각 채널에서 수집된 영상 개수
  5. 완료 후 CSV 다운로드 버튼으로 결과 파일을 내려받습니다.

CSV 결과

  • 주요 컬럼:
    • 주차 (2025년 52주차 등)
    • 채널명
    • 플랫폼 (유튜브)
    • 날짜 / 시간
    • 영상 제목
    • 영상 URL (http 링크) – Excel에서 자동으로 클릭 가능한 링크로 인식
    • 조회수 / 좋아요 수 / 댓글 수
    • 영상 유형 (예: 게임 플레이, 하이라이트 등)
  • 주의:
    • 예전 버전에서는 HYPERLINK() 함수를 써서 컬럼이 밀리는 문제가 있었지만,
      현재 버전은 일반 URL만 출력해서 컬럼이 정확히 맞습니다.

에러 및 할당량 안내

  • 채널별 에러는 화면 하단에 오류 목록으로 표시됩니다.
    • 예: 채널 ID를 찾을 수 없습니다, YouTube API 할당량 초과 등
  • 대표적인 상황:
    • 잘못된 채널 ID / URL:
      • 해당 채널만 건너뛰고, 나머지 채널은 계속 처리합니다.
      • CSV에는 정상 채널만 포함되며, 오류 채널은 화면의 오류 상세에서 확인합니다.
    • YouTube API 할당량 초과:
      • 더 이상 수집을 진행하지 않고, 남은 채널은 모두 에러로 표기됩니다.
      • 화면 상단에 할당량 관련 안내 메시지가 표시됩니다.

할당량(Quota) 절약 팁

  • UC로 시작하는 채널 ID를 엑셀에 직접 적어두면,
    • 채널 검색용 API를 쓰지 않아도 되어 할당량을 크게 절약할 수 있습니다.
  • 같은 채널을 여러 번 돌려야 할 일이 있으면,
    • 처음 한 번만 URL로 ID를 찾아보고,
    • 이후에는 그 ID를 엑셀에 고정해서 사용하는 것을 권장합니다.

💡 에이전트 가이드

  • 영상 집계가 어긋나 보이면, 먼저 CSV 컬럼 순서와 채널 ID 입력 상태를 확인하세요.
  • 특정 채널만 계속 오류가 난다면:
    • 엑셀에서 그 채널의 channelId 또는 youtubeUrl이 올바른지 다시 확인한 뒤,
    • 수정 후 다시 아카이빙을 실행하면 됩니다.

5.9 에이전트 성과 분석 (관리자 전용)

에이전트 성과 분석이란? 에이전트별 이슈 처리 통계와 성과를 분석하는 기능입니다.

접근 방법:

  1. 상단 메뉴 → 관리 → 데이터 관리 → 에이전트 성과 탭
  2. 또는 Admin 페이지에서 "에이전트 성과" 탭 클릭

분석 항목:

  • 처리 통계: 총 처리 건수, 심각도별 처리 건수
  • 감정 분석: 긍정/부정/중립 이슈 처리 건수
  • 상태별 통계: 완료, 진행중, 분류 완료, 열림 상태별 건수
  • 처리 시간: 평균, 중간값, 최소, 최대 처리 시간
  • 카테고리별 통계: 카테고리별 처리 건수
  • 프로젝트별 통계: 프로젝트별 처리 건수
  • 일별 통계: 날짜별 처리 건수 및 평균 처리 시간

필터 옵션:

  • 에이전트: 특정 에이전트만 선택
  • 프로젝트: 특정 프로젝트만 선택
  • 기간: 시작일과 종료일 선택 (기본: 최근 7일)

Excel 내보내기:

  • "Excel 내보내기" 버튼을 클릭하여 통계를 Excel 파일로 다운로드
  • 모든 통계 데이터가 포함됩니다

KST 기준:

  • 모든 날짜 데이터는 한국 시간(KST) 기준으로 표시됩니다

5.10 업무 가이드 관리 (관리자 전용)

업무 가이드란? 에이전트가 업무를 수행할 때 참고할 수 있는 가이드 문서입니다. AI 어시스턴트(RAG) 검색에도 사용됩니다.

접근 방법:

  1. 상단 ☰ 메뉴 → 가이드 관리(ADMIN / LEAD / SUPERADMIN)
  2. 우측 상단 설명서 링크는 매뉴얼 #guide-management 로 이동합니다

화면 버튼 요약: 13.10 업무 가이드 관리(Guide Management) 화면 참고.

파일 업로드(일괄):

  1. 파일 업로드를 클릭합니다
  2. PDF·DOCX·TXT·MD·HTML 등을 선택합니다(다중 선택 가능)
  3. 제출 시 서버가 텍스트를 추출하고 가이드를 생성·임베딩합니다(autoSplit 등 서버 설정에 따름)

수동 등록:

  1. 새 가이드로 제목·본문·유형(general 등)·카테고리·태그·출처 URL을 입력합니다
  2. 만들기로 저장합니다

검색·필터:

  • 검색창: 제목·본문·카테고리·태그
  • 유형 드롭다운으로 가이드 타입 필터

매뉴얼 본문을 RAG에 넣기: 관리자가 cd backend && node scripts/import-user-manual-to-rag.js --replace --lang=all 실행 시 USER_MANUAL.md(한국어)와 USER_MANUAL.en.md(영어) 출처 가이드가 재등록됩니다(섹션 13 포함). 영어만/한국어만 갱신할 때는 --lang=en 또는 --lang=ko 를 사용합니다.

5.11 업무 체크리스트

업무 체크리스트란? 에이전트가 업무 중 실행한 항목을 날짜별로 체크할 수 있는 기능입니다. 관리자가 등록한 항목이 선택한 날짜·작업 구분·요일에 맞게 표시됩니다.

5.11.1 에이전트: 업무 체크리스트

접근 방법:

  1. 메인 화면 상단 메뉴를 클릭합니다
  2. **"업무 체크리스트"**를 선택합니다

사용 방법:

  1. 날짜를 선택합니다 (기본: 오늘). 과거·미래 날짜도 선택 가능합니다.
  2. 작업 구분을 선택합니다 (전체, 주간, 오후, 야간, 정오). 본인 근무에 맞게 선택하면 해당 항목만 표시됩니다.
  3. 표시된 항목에 대해 실행한 경우 체크박스를 선택합니다. 체크 시 자동 저장됩니다.
  4. 완료한 항목 수가 하단에 "완료: n / m 항목"으로 표시됩니다.

표시 규칙:

  • 관리자가 설정한 작업 구분(주간/오후/야간/정오)에 맞는 항목만 표시됩니다. "전체"는 모든 작업 구분에 표시됩니다.
  • 적용 기간(시작일~종료일) 내 날짜에만 표시됩니다.
  • 적용 월이 지정된 항목은 해당 월에만 표시됩니다.
  • 적용 요일이 지정된 항목은 선택한 날짜의 요일이 해당 요일일 때만 표시됩니다. (예: 월·수·금만 지정된 항목은 화·목·토·일에는 표시되지 않습니다.)
  • 요일을 지정하지 않은 항목은 모든 요일에 표시됩니다.

5.11.2 관리자: 업무 체크리스트 관리

접근 방법:

  1. 메인 화면 상단 메뉴를 클릭합니다 (관리자/리드/슈퍼관리자만 표시)
  2. **"업무 체크리스트 관리"**를 선택합니다

항목 추가:

  1. "+ 항목 추가" 버튼을 클릭합니다
  2. 다음 정보를 입력합니다:
    • 항목 이름: 예) 일일 리포트 제출
    • 작업 구분: 전체, 주간, 오후, 야간, 정오 중 선택
    • 시작일/종료일 (선택): 적용 기간. 비워두면 기간 제한 없음
    • 적용 월 (선택): 1월~12월 중 표시할 월 선택. 비워두면 매월 표시
    • 적용 요일 (선택): 월~일 중 표시할 요일 선택. 비워두면 모든 요일에 표시
    • 활성화: 체크 시 에이전트 화면에 표시
  3. 저장 버튼을 클릭합니다

항목 수정/삭제:

  • 목록에서 수정 버튼으로 내용 변경, 삭제 버튼으로 항목 제거
  • ↑/↓ 버튼으로 표시 순서를 변경할 수 있습니다 (저장 후 목록 새로고침)

필터:

  • 비활성 항목 포함: 비활성화된 항목도 목록에 표시
  • 작업 구분: 특정 작업 구분의 항목만 목록에 표시

5.12 LINE 알림 관리 (관리자 전용)

LINE 알림이란? 지정된 시간에 LINE 메시지로 업무 알림을 전송하는 기능입니다.

접근 방법:

  1. 메인 화면 상단의 "🔔 업무 알림 관리" 버튼을 클릭합니다 (관리자만 표시)
  2. 업무 알림 관리 페이지로 이동합니다

알림 등록:

  1. "알림 등록" 버튼을 클릭합니다
  2. 다음 정보를 입력합니다:
    • 업무명
    • 반복 타입 (매일, 매주, 매월, 특정 날짜, 간격)
    • 알림 시간
    • LINE 채널 ID
    • 메시지 (선택)
  3. "저장" 버튼을 클릭합니다

반복 타입:

  • 매일: 지정된 기간 동안 매일 동일 시간에 전송
  • 매주: 지정된 요일에 전송
  • 매월: 지정된 일자에 전송
  • 특정 날짜: 한 번만 전송
  • 간격: 지정된 시간 범위 내에서 일정 간격으로 반복 전송

알림 복사:

  1. 기존 알림의 "복사" 버튼을 클릭합니다
  2. 시간만 변경하여 새 알림으로 등록할 수 있습니다

반복 등록:

  • 하루에 여러 번 동일한 알림을 등록할 수 있습니다
  • "시간 추가" 버튼으로 추가 시간을 입력합니다

6. AI 어시스턴트

6.1 AI 어시스턴트란?

AI 어시스턴트는 업무 가이드 문서를 기반으로 질문에 답변해주는 지능형 챗봇입니다.

주요 특징:

  • RAG (Retrieval-Augmented Generation): 업로드된 가이드 문서를 검색하여 정확한 답변 제공
  • 의미 기반 검색: 키워드가 아닌 의미를 이해하여 관련 문서 찾기
  • 대화형 인터페이스: 채팅 형태로 자연스러운 질의응답
  • 관련 문서 제시: 질문과 관련된 여러 가이드 문서를 함께 제공

6.2 사용 방법

1단계: AI 어시스턴트 접속

  1. 메인 화면 상단의 "🤖 AI 어시스턴트" 버튼을 클릭합니다
  2. AI 어시스턴트 페이지로 이동합니다

2단계: 질문 입력

  1. 하단의 입력 필드에 질문을 입력합니다
    • 예: "일일 리포트 작성 방법은?"
    • 예: "이슈 처리 절차는?"
    • 예: "SLA 위반 시 어떻게 해야 하나요?"
  2. Enter 키를 누르거나 전송 버튼을 클릭합니다

3단계: 답변 확인

  • AI가 업무 가이드 문서를 검색하여 답변을 제공합니다
  • 관련 가이드 문서가 있으면 함께 표시됩니다
  • 답변 하단에 참고한 문서 목록이 표시됩니다

6.3 효과적인 질문 방법

좋은 질문 예시:

  • ✅ "일일 리포트는 어떻게 작성하나요?"
  • ✅ "이슈 처리 시 주의사항은 무엇인가요?"
  • ✅ "SLA 위반 이슈는 어떻게 처리하나요?"

나쁜 질문 예시:

  • ❌ "리포트" (너무 짧음)
  • ❌ "어떻게 해?" (너무 모호함)

💡 팁: 질문을 구체적이고 명확하게 작성할수록 더 정확한 답변을 받을 수 있습니다.

6.4 가이드 문서 관리

가이드 문서 업로드 (관리자 전용):

  1. "📚 가이드 관리" 페이지로 이동합니다
  2. PDF 파일을 업로드합니다
  3. 파일이 자동으로 파싱되고 임베딩 처리됩니다
  4. AI 어시스턴트가 해당 가이드를 기반으로 답변합니다

가이드 검색:

  • 가이드 관리 페이지에서 제목, 내용, 태그로 검색 가능
  • 카테고리별로 필터링 가능

7. 공지사항 관리

7.1 공지사항 페이지

공지사항 페이지란? 모든 공지사항을 한 곳에서 관리하고 확인할 수 있는 전용 페이지입니다.

접근 방법:

  1. 메인 화면 상단의 "📢 공지사항" 버튼을 클릭합니다
  2. 공지사항 페이지로 이동합니다

7.2 공지사항 필터링

필터 옵션:

  • 날짜별: 특정 날짜의 공지사항만 표시
    • 날짜 선택기를 사용하여 날짜 선택
  • 주차별: 특정 주차의 공지사항만 표시
    • 주차 선택기를 사용하여 주차 선택 (예: 2025-W52)
  • 담당자별: 특정 담당자의 공지사항만 표시
    • 드롭다운에서 담당자 선택
  • 게임별: 특정 게임의 공지사항만 표시
    • 드롭다운에서 게임 선택
  • 검색: 제목이나 내용으로 검색
    • 검색 필드에 키워드 입력

필터 조합:

  • 여러 필터를 동시에 사용할 수 있습니다
  • 예: "2025-12-20" + "PUBG PC" → 2025년 12월 20일 PUBG PC 공지사항만 표시

7.3 공지사항 열람 확인

열람 상태 확인:

  • 각 공지사항에 열람한 에이전트 목록이 표시됩니다
  • 열람하지 않은 에이전트 목록도 확인할 수 있습니다
  • 열람 시간도 함께 표시됩니다

열람 표시:

  1. 공지사항을 확인합니다
  2. "확인" 버튼을 클릭합니다
  3. 본인이 열람했다고 표시됩니다
  4. 열람 시간이 기록됩니다

열람 정보:

  • 공지사항 상세에서 열람한 에이전트 목록 확인 가능
  • 열람 시간 확인 가능
  • 누가 아직 읽지 않았는지 확인 가능

7.4 공지사항 작성 (관리자 전용)

작성 방법:

    • 게임명
    • 담당자 이름
    • 카테고리
    • 공지 내용
    • 공지 날짜

8. 데스크톱 알림

8.1 데스크톱 알림이란?

데스크톱 알림은 브라우저 밖 데스크톱에 알림을 표시하는 기능입니다. 브라우저를 닫아도 알림을 받을 수 있습니다.

알림 타입:

  • 새 이슈 알림: 새 이슈가 생성될 때
  • SLA 위반 알림: SLA 위반 시
  • 이슈 업데이트 알림: 이슈 상태가 변경될 때 (선택적)

8.2 알림 설정

접근 방법:

  1. 메인 화면 상단의 "🔔 알림 설정" 버튼을 클릭합니다
  2. 알림 설정 페이지로 이동합니다

권한 요청:

  1. "알림 권한 요청" 버튼을 클릭합니다
  2. 브라우저에서 알림 권한을 허용합니다
  3. 권한이 허용되면 "✅ 허용됨"으로 표시됩니다

⚠️ 주의: 권한이 거부된 경우 브라우저 설정에서 수동으로 허용해야 합니다.

알림 설정 항목:

  • 데스크톱 알림 활성화: 전체 알림 기능 켜기/끄기
  • 알림 소리: 알림 표시 시 소리 재생 여부
  • 새 이슈 알림: 새 이슈 생성 시 알림 표시
  • SLA 위반 알림: SLA 위반 시 알림 표시
  • 이슈 업데이트 알림: 이슈 상태 변경 시 알림 표시

설정 저장:

  • 설정은 자동으로 저장됩니다
  • 브라우저를 닫아도 설정이 유지됩니다

8.3 알림 사용하기

알림 표시:

  • 새 이슈가 생성되면 자동으로 데스크톱 알림이 표시됩니다
  • SLA 위반 시 자동으로 알림이 표시됩니다
  • 알림에는 이슈 제목과 심각도가 표시됩니다

알림 클릭:

  • 알림을 클릭하면 해당 이슈로 자동 이동합니다
  • 메인 화면으로 전환되고 이슈 상세 패널이 열립니다
  • 브라우저가 닫혀있어도 자동으로 열립니다

알림 자동 닫기:

  • 알림은 5초 후 자동으로 닫힙니다
  • Sev1 이슈 알림은 사용자가 직접 닫을 때까지 표시됩니다

8.4 소리 알림

소리 알림:

  • 알림 표시 시 beep 소리가 재생됩니다
  • 설정에서 소리 알림을 켜기/끄기할 수 있습니다
  • 브라우저가 음소거 상태면 소리가 재생되지 않습니다

9. 보고서 생성

6.1 일일 보고서 다운로드

일일 보고서란? 하루 동안 수집된 이슈를 정리한 Excel 보고서입니다.

다운로드 방법:

  1. 상단 메뉴에서 **"일일 보고서"**를 클릭합니다
  2. 시작 날짜와 종료 날짜를 선택합니다
  3. 프로젝트를 선택합니다 (선택 사항)
  4. "보고서 다운로드" 버튼을 클릭합니다
  5. Excel 파일이 다운로드됩니다

보고서 내용:

  • 요약 통계
  • VOC (Voice of Customer) 데이터
  • 이슈 목록
  • 원본 데이터

💡 팁: 보고서는 날짜 범위 내의 모든 이슈를 포함합니다 (개수 제한 없음).

6.2 주간 보고서 생성

주간 보고서란? 일주일 동안의 이슈를 종합 분석한 보고서입니다.

생성 방법:

  1. 상단 메뉴에서 **"주간 보고서"**를 클릭합니다
  2. 기간 선택에서 시작일과 종료일을 선택합니다
  3. "보고서 생성" 버튼을 클릭합니다
  4. 생성된 보고서를 Excel 파일로 다운로드할 수 있습니다

6.3 PUBGM 주간보고서 생성

PUBGM 주간보고서란? PUBG MOBILE 일일 VoC(고객의 소리) 보고서 엑셀을 기반으로, 주간 단위 모니터링 보고서 엑셀을 생성하는 기능입니다. 성향별·이슈별 동향, 주간 동향 상세 요약, LLM 동향 요약(선택), 주요 이슈 증감 등이 포함된 시트가 생성됩니다.

사용 흐름: 소스 등록 → 기간 설정 → 산출물 생성 → 다운로드

  1. 주간 보고서 메뉴에서 "PUBGM 주간보고서 생성" 카드를 확인합니다.

  2. 소스 파일 등록

    • 일일 VoC 보고서 엑셀(.xlsx, .xls)을 준비합니다. 시트명은 반드시 VoC이며, 날짜·대분류·중분류·성향·내용·요약·건수 등 컬럼이 있어야 합니다.
    • 파일 선택 후 업로드를 클릭합니다.
    • 업로드된 파일 목록에서 **"이 소스로 산출물 생성"**으로 사용할 소스를 선택할 수 있습니다. 삭제로 소스를 제거할 수 있습니다.

  3. 보고 기간 설정

    • 주간(자동): VoC 데이터의 최신일 기준 직전 주(월요일~일요일)로 자동 계산됩니다.
    • 기간 지정: 시작일과 종료일을 직접 선택합니다. 시작일은 종료일보다 늦을 수 없습니다.

  4. 산출물 생성

    • 소스를 하나 선택한 뒤 **"이 소스로 산출물 생성"**을 클릭합니다.
    • 산출물 목록 새로고침으로 생성된 파일 목록을 확인합니다.
    • 파일명 형식: PUBGMOBILE_모니터링_주간보고서_YYYYMMDD_YYYYMMDD.xlsx

  5. 다운로드 및 삭제

    • 목록에서 다운로드를 클릭하면 생성된 주간보고서 엑셀을 받을 수 있습니다.
    • 삭제를 클릭하면 해당 산출물 파일이 서버에서 제거됩니다.

산출물 엑셀 시트 구성 (요약)

  • 성향별 주간 동향 수, 이슈 별 동향 수, 주간 동향 상세 요약, 주요 이슈 증감, 주간_범위_정보 등이 포함됩니다.
  • 동향 요약 (LLM) 시트는 OPENAI_API_KEY가 설정된 경우에만 생성되며, 부정/긍정 동향을 항목별 요약문으로 보여줍니다.

주의사항

  • 소스 엑셀의 첫 번째 시트명은 VoC(대소문자 구분)여야 합니다.
  • VoC 시트에 유효한 데이터(날짜, 대분류 등)가 없으면 생성이 실패할 수 있습니다. 오류 시 서버 로그의 [VocParse] 메시지를 확인하세요.

10. 문제 해결

7.1 로그인 문제

문제: 로그인이 안 됩니다.

해결 방법:

  1. 이메일과 비밀번호를 다시 확인합니다
  2. 대소문자, 특수문자를 정확히 입력했는지 확인합니다
  3. 관리자에게 계정 상태를 확인 요청합니다
  4. 브라우저 캐시를 삭제하고 다시 시도합니다 (Ctrl+Shift+Delete)

7.2 이슈가 표시되지 않음

문제: 이슈 목록에 아무것도 표시되지 않습니다.

해결 방법:

  1. 프로젝트 선택이 올바른지 확인합니다
    • "전체 프로젝트"로 변경해보세요
  2. 필터를 확인합니다
    • 필터를 모두 해제해보세요
  3. 검색어를 지워봅니다
  4. 브라우저를 새로고침합니다 (F5)
  5. 다른 브라우저에서 시도해봅니다

7.3 수동 수집 실패

문제: 수동 수집이 실패합니다.

해결 방법:

  1. URL이 올바른지 확인합니다
    • cafe.naver.com 도메인인지 확인
    • URL이 완전한지 확인
  2. URL이 공개 게시글인지 확인합니다
    • 비공개 게시글은 수집할 수 없습니다
  3. 네트워크 연결을 확인합니다
  4. 잠시 후 다시 시도합니다
  5. 관리자에게 문의합니다

7.4 이미지가 표시되지 않음

문제: 이슈의 스크린샷 이미지가 표시되지 않습니다.

해결 방법:

  1. 브라우저를 새로고침합니다 (F5)
  2. 이미지 URL이 올바른지 확인합니다 (개발자 도구 F12 → Network 탭)
  3. 브라우저 캐시를 삭제합니다
  4. 다른 브라우저에서 시도해봅니다

7.5 실시간 업데이트가 안 됨

문제: 이슈 목록이 실시간으로 업데이트되지 않습니다.

해결 방법:

  1. 브라우저를 새로고침합니다 (F5)
  2. 네트워크 연결을 확인합니다
  3. WebSocket 연결 상태를 확인합니다 (개발자 도구 F12 → Network → WS)
    • 정상 연결 시: ws://127.0.0.1:8080 (개발 환경) 또는 ws://your-server:8080 (프로덕션)
    • 연결 실패 시: 네트워크 연결 및 백엔드 서버 상태를 확인하세요
  4. 다른 브라우저에서 시도해봅니다

7.6 상태 변경이 저장되지 않음

문제: 이슈 상태를 변경했는데 저장되지 않습니다.

해결 방법:

  1. 네트워크 연결을 확인합니다
  2. 브라우저를 새로고침합니다 (F5)
  3. 다시 시도합니다
  4. 문제가 계속되면 관리자에게 문의합니다

11. FAQ

Q1. 이슈는 어떻게 자동으로 수집되나요?

A: 네이버 카페 크롤러와 슬랙 워커가 주기적으로 실행되어 새 게시글을 자동으로 수집합니다. 크롤러는 약 5분마다 실행되며, 새 게시글이 발견되면 자동으로 이슈로 등록됩니다.

Q2. AI 분류는 어떻게 작동하나요?

A: OpenAI API를 사용하여 이슈 내용을 분석하고 자동으로 카테고리와 심각도를 분류합니다. 분류 결과와 이유는 이슈 상세 패널에서 확인할 수 있습니다. AI 분류가 실패하면 규칙 기반 분류로 자동 전환됩니다.

Q3. SLA는 어떻게 계산되나요?

A: 각 심각도별로 설정된 SLA 정책에 따라 마감 시간이 계산됩니다. 예를 들어, Sev1 이슈는 1시간 내 처리해야 하며, 이 시간이 지나면 KPI 대시보드에 "SLA 임박"으로 표시됩니다.

Q4. 여러 이슈를 한 번에 처리할 수 있나요?

A: 네, 이슈 목록에서 여러 이슈를 선택하고 "선택 완료" 또는 "선택 보고서 제외" 버튼을 클릭하여 일괄 처리할 수 있습니다.

Q5. 보고서는 언제 생성되나요?

A: 일일 보고서는 원하는 날짜 범위를 선택하여 수동으로 다운로드할 수 있습니다. 보고서는 날짜 범위 내의 모든 이슈를 포함합니다 (개수 제한 없음).

Q6. 에이전트 상태는 어떻게 업데이트되나요?

A: 에이전트가 시스템에 로그인하면 자동으로 온라인 상태로 변경됩니다. 일정 시간 동안 활동이 없으면 자리비움 상태로 변경됩니다. 상태는 실시간으로 업데이트됩니다.

Q7. 수동 수집은 언제 사용하나요?

A: 크롤러가 놓친 게시글이나 긴급하게 수집해야 하는 게시글을 즉시 수집할 때 사용합니다. 수동 수집은 약 10-30초가 소요됩니다.

Q8. 이슈를 삭제할 수 있나요?

A: 일반 사용자(AGENT)는 이슈를 삭제할 수 없습니다. 이슈를 보고서에서 제외하고 싶다면 "선택 보고서 제외" 기능을 사용하세요. 이슈는 목록에 계속 표시되지만 보고서 생성 시 제외됩니다.

Q9. 프로젝트를 선택하지 않으면 어떻게 되나요?

A: "전체 프로젝트"를 선택하면 모든 프로젝트의 이슈가 표시됩니다. 특정 프로젝트를 선택하면 해당 프로젝트의 이슈만 표시됩니다.

Q10. 이슈는 최대 몇 개까지 표시되나요?

A: 메인 화면의 이슈 큐에는 최대 1000개까지 표시됩니다. 하지만 보고서에는 날짜 범위 내의 모든 이슈가 포함됩니다 (개수 제한 없음).

Q11. 스크린샷은 얼마나 보관되나요?

A: 스크린샷은 10일간 보관되며, 10일이 지나면 자동으로 삭제됩니다. 매일 자정에 자동 정리가 실행됩니다.

Q12. 댓글은 언제 자동으로 수집되나요?

A: 다음 조건 중 하나를 만족하면 댓글이 자동으로 수집됩니다:

  • 게시글 작성자가 특정 운영진 계정인 경우 (기본: GM네로, PUBG운영우진 등)
  • 댓글 수가 30개 이상인 경우 (기본 임계값)

수집된 댓글은 이슈 상세 패널에서 확인할 수 있습니다.

Q13. AI 어시스턴트는 어떻게 사용하나요?

A:

  1. 메인 화면 상단의 "🤖 AI 어시스턴트" 버튼을 클릭합니다
  2. 질문을 입력하고 Enter를 누릅니다
  3. AI가 업무 가이드 문서를 검색하여 답변을 제공합니다
  4. 관련 가이드 문서가 있으면 함께 표시됩니다

Q14. 데스크톱 알림이 작동하지 않아요.

A:

  1. 알림 설정 페이지에서 권한이 허용되었는지 확인합니다
  2. 브라우저 설정에서 알림 권한을 확인합니다
  3. 알림 설정에서 "데스크톱 알림 활성화"가 켜져있는지 확인합니다
  4. 브라우저를 재시작해봅니다

Q15. 공지사항은 어디서 확인하나요?

A:

  1. 메인 화면 상단의 "📢 공지사항" 버튼을 클릭합니다
  2. 공지사항 페이지에서 날짜, 주차, 담당자, 게임별로 필터링할 수 있습니다
  3. 각 공지사항의 "확인" 버튼을 클릭하여 열람 표시를 할 수 있습니다

Q16. LINE 알림은 어떻게 설정하나요?

A:

  • 관리자 권한이 필요합니다
  • "🔔 업무 알림 관리" 페이지에서 알림을 등록합니다
  • 반복 타입(매일, 매주, 매월 등)과 시간을 설정합니다
  • LINE 채널 ID를 입력합니다

Q17. 에이전트 성과는 어디서 확인하나요?

A:

  • 관리자 권한이 필요합니다
  • 상단 메뉴 → 관리 → 데이터 관리 → 에이전트 성과 탭
  • 기간, 에이전트, 프로젝트별로 필터링하여 통계를 확인할 수 있습니다
  • Excel로 내보내기 가능합니다

Q18. 모든 날짜는 어떤 시간 기준인가요?

A:

  • 모든 날짜 데이터는 한국 시간(KST, Asia/Seoul) 기준입니다
  • 이슈 생성 시간, 처리 시간, 통계 날짜 등 모든 시간이 KST로 표시됩니다

Q19. 업무 체크리스트는 어디서 보나요?

A:

  1. 메인 화면 상단 메뉴를 클릭한 뒤 **"업무 체크리스트"**를 선택합니다
  2. 날짜와 작업 구분(주간/오후/야간/정오 등)을 선택하면 해당 날짜·요일·작업에 맞는 항목만 표시됩니다
  3. 관리자는 메뉴 → **"업무 체크리스트 관리"**에서 항목을 추가·수정·삭제하고, 적용 요일(월~일)을 지정할 수 있습니다

12.4 포럼 모니터링 화면

데이터: 포럼 모니터링 서비스가 생성한 최신 일일 리포트 1건 (/v1/reports/daily/latest).

구역:

  • Hot topics: 상대적으로 주목도가 높은 토픽과 점수(스냅샷 기반; 증분이 없으면 fallback_absolute 등 표기가 붙을 수 있음)
  • New topics: 최근 24시간 신규로 잡힌 토픽(시스템 first_seen 기준), 제목 클릭 시 포럼 원문
  • Category summary: Discourse 카테고리 ID별 신규 수·핫 점수 합
  • 새로고침 / API 문서: 데이터 재요청, OpenAPI(Swagger) 새 탭

개발 환경: 브라우저는 보통 /forum-api로 요청하고, Vite 개발 서버가 포럼 API(기본 포트 9090)로 넘깁니다. 화면이 비거나 오류면 포럼 서비스·DB 기동 여부를 관리자에게 확인하세요.

12.5 이슈 상세와 Discourse(포럼) 글

  • inZOI Forums 등 Discourse에서 생성된 이슈는 본문에 메타가 있으면 대표 이미지를 상세 영역에서 볼 수 있습니다(네이버 카페 스크린샷과 병행).

12.6 상단 글로벌 내비게이션과 혼동하지 않기

  • Monitoring (상단 탭, ADMIN/LEAD): 모니터링 제어 페이지(크롤 트리거, 키워드 등) — 포럼 모니터링과 이름만 비슷하고 다른 화면입니다.
  • Daily report: 일일 리포트 다운로드/생성 등 별도 워크플로입니다.
  • Board statistics 등: 게시판 통계·스냅샷 등, 역시 포럼 동향 카드 UI와는 별도입니다.
