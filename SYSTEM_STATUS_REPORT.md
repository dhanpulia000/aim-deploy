## 3. 시스템 아키텍처

### 3.1 기술 스택

#### 백엔드
- **런타임**: Node.js
- **프레임워크**: Express.js
- **데이터베이스**: SQLite (better-sqlite3)
- **ORM**: Prisma (스키마 정의용, 실제 쿼리는 better-sqlite3 직접 사용)
- **웹 크롤링**: Playwright
- **인증**: JWT (jsonwebtoken)
- **실시간 통신**: WebSocket (ws)
- **기타 주요 라이브러리**:
  - axios (HTTP 클라이언트)
  - discord.js (Discord 통합)
  - @slack/web-api (Slack 통합)
  - exceljs, xlsx (Excel 처리)
  - google-spreadsheet (Google Sheets 통합)

#### 프론트엔드
- **프레임워크**: React 18.2.0
- **빌드 도구**: Vite 5.0.8
- **언어**: TypeScript
- **스타일링**: Tailwind CSS
- **차트**: Recharts
- **가상화**: react-virtuoso (대용량 리스트 렌더링)
- **마크다운**: markdown-it

### 3.2 프로젝트 구조

```
AIM/
├── backend/                 # 백엔드 서버
│   ├── server.js           # 서버 진입점
│   ├── app.js              # Express 앱 설정
│   ├── controllers/        # API 컨트롤러
│   ├── services/           # 비즈니스 로직
│   ├── workers/            # 백그라운드 워커
│   │   ├── monitoring/     # 모니터링 워커 (크롤러)
│   │   └── ingestion/     # 데이터 수집 워커
│   ├── routes/             # API 라우트
│   ├── libs/               # 유틸리티 라이브러리
│   ├── prisma/             # 데이터베이스 스키마
│   └── utils/              # 헬퍼 함수
├── src/                    # 프론트엔드 소스
│   ├── App.tsx             # 메인 앱 컴포넌트
│   ├── components/         # React 컴포넌트
│   ├── contexts/           # React Context
│   ├── hooks/              # 커스텀 훅
│   └── utils/              # 유틸리티
└── dist/                   # 빌드 결과물
```

## 4. 주요 기능

### 4.1 이슈 관리 시스템

#### 핵심 기능
- ✅ **이슈 수집**: 네이버 카페 게시글 자동 크롤링

- ✅ **에이전트 할당**: 자동 할당 (근무 시간 기반) 및 수동 할당
- ✅ **프로젝트별 필터링**: 다중 프로젝트 지원
- ✅ **검색 기능**: 제목, 본문, 카테고리 검색
- ✅ **상태 필터링**: 심각도, 상태, 카테고리별 필터

#### 최근 개선 사항

### 4.2 크롤링 시스템

#### 네이버 카페 크롤러
- **워커**: `naverCafe.worker.js`
- **스캔 간격**: 기본 180초 (3분)
- **기능**:

## 5. 데이터베이스 스키마

### 5.1 주요 테이블

#### ReportItemIssue (이슈)
- **필드**: id, summary, detail, status, severity, categoryId, projectId, assignedAgentId, sourceCreatedAt 등
- **인덱스**: projectId, status, severity, createdAt, assignedAgentId

#### Agent (에이전트)
- **필드**: id, name, status, projectId, isActive, channelFocus 등
- **관계**: ReportItemIssue (assignedAgentId)

#### AgentSchedule (에이전트 스케줄)
- **필드**: id, agentId, scheduleType, dayOfWeek, specificDate, startTime, endTime, workType
- **관계**: Agent (agentId)

#### MonitoredBoard (모니터링 게시판)
- **필드**: id, name, url, cafeGame, isActive, lastScanAt, lastArticleId 등
- **관계**: Project (projectId)

#### RawLog (원시 로그)
- **필드**: id, source, content, author, timestamp, isProcessed, metadata
- **용도**: 크롤러가 수집한 원시 데이터 저장

#### CategoryGroup / Category (카테고리)
- **계층 구조**: 대분류 → 중분류
- **프로젝트별 관리**: projectId로 구분
