# 캘린더 이벤트 등록 로직 분석

## 1. 전체 흐름

```
[프론트엔드] handleSaveEvent
    → fetch POST /api/calendar/events
    → [Vite Proxy 5173→8080]
    → [백엔드] authenticate → requireRole → createEvent
    → DB INSERT → 201 응답
    → loadEvents() 재호출
```

## 2. 프론트엔드 (Calendar.tsx)

### 2.1 handleSaveEvent 진입 조건
- `token` 존재 (useAuth)
- `isAdmin` (ADMIN/LEAD/SUPERADMIN)
- `!saving` (중복 방지)
- `startDate`, `title.trim()` 필수

### 2.2 요청 페이로드
```json
{
  "platform": "PC" | "MO",
  "startDate": "2026-02-13T09:00:00+09:00",
  "endDate": "2026-02-13T18:00:00+09:00",
  "title": "테스트",
  "link": null | "https://..."
}
```

### 2.3 잠재적 이슈
| 항목 | 확인 | 비고 |
|------|------|------|
| link.trim() | ⚠️ | `link`가 undefined면 `link.trim()` → TypeError |
| token | ✅ | AuthContext에서 localStorage "authToken" |
| fetch URL | ✅ | 상대경로 `/api/calendar/events` → 프록시 통해 8080 |

## 3. 백엔드 (calendar.controller.js)

### 3.1 createEvent 검증
- platform: 'PC' 또는 'MO'
- startDate, endDate 필수
- title: 문자열, trim 후 비어있으면 안됨
- `new Date(startDate) > new Date(endDate)` → 422

### 3.2 DB INSERT
```sql
INSERT INTO CalendarEvent (platform, startDate, endDate, title, link, updatedAt)
VALUES (?, ?, ?, ?, ?, datetime('now'))
```

### 3.3 응답
- 201 Created
- `{ success: true, data: { event }, message: "Event created successfully" }`

## 4. 라우트/미들웨어 체인

```
POST /api/calendar/events
  → app.use('/api', validateResponse)  // res.json 래핑만, 요청 차단 없음
  → app.use('/api', apiRoutes)
  → router.use('/calendar', calendarRoutes)
  → authenticate (JWT 검증)
  → requireRole(['ADMIN','LEAD','SUPERADMIN'])
  → createEvent
```

## 5. 가능한 실패 원인

### A. link undefined로 인한 TypeError
`link`가 undefined일 때 `link.trim()` 호출 시 크래시 → catch로 가지만, 에러 메시지가 불명확할 수 있음.

### B. 토큰 만료/불일치
- 브라우저 localStorage의 token이 만료되었을 수 있음
- 401 시 "로그인이 만료되었습니다" alert 표시됨

### C. 프록시/네트워크
- 백엔드 8080 미기동 시 fetch 실패 → catch
- Vite proxy 설정: `/api` → `http://127.0.0.1:8080`

### D. loadEvents 날짜 범위
- getMonthRange(): 3개월 전 ~ 4개월 후
- 새 이벤트가 이 범위 안이면 조회됨

### E. DB 테이블 미존재
- migration 013_add_calendar_event.sql
- 서버 기동 시 자동 적용

## 6. 실행 환경 확인

**캘린더 이벤트 등록이 동작하려면 백엔드가 실행 중이어야 합니다.**

- **개발**: `npm run dev` (프론트 5173) + 백엔드 8080 동시 실행
- **백엔드 실행**: `cd backend && node server.js` 또는 `npm run start:pm2`
- **API 직접 테스트**: `node scripts/test-calendar-event.js admin@example.com admin123`

백엔드가 꺼져 있으면 "서버에 연결할 수 없습니다" 알림이 표시됩니다.
