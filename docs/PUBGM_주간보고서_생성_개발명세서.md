# PUBGM 주간보고서 생성 세션 — 개발명세서

## 1. 개요

### 1.1 세션 정보
| 항목 | 값 |
|------|-----|
| **세션 명칭** | PUBGM 주간보고서 생성 |
| **메뉴 경로** | 주간보고서 (`/weekly-report`) |
| **위치** | 주간보고서 페이지 내 "PUBGM 주간보고서 생성" 카드 |

### 1.2 목적
- **소스**: PUBG MOBILE 일일 VoC 보고서 엑셀 파일 (.xlsx)
- **산출물**: 주간 모니터링 보고서 엑셀 (성향별·이슈별 동향 집계 포함)
- **흐름**: 소스 파일 등록 → 보고 기간 설정 → 산출물 생성 → 다운로드

### 1.3 관련 파일
| 구분 | 경로 |
|------|------|
| 프론트엔드 | `src/WeeklyReportGenerator.tsx` |
| 백엔드 컨트롤러 | `backend/controllers/reports.controller.js` |
| VoC 보고서 생성 서비스 | `backend/services/weeklyVocReportFromExcel.service.js` |
| 기술명세서(원본) | `docs/NODEJS_기술명세서.md` |

## 3. API 명세

### 3.1 소스 관리

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/reports/weekly-sources` | 소스 파일 목록 조회 |
| POST | `/api/reports/weekly-sources/upload` | 소스 파일 업로드 (multipart/form-data, field: `file`) |
| DELETE | `/api/reports/weekly-sources/:sourceId` | 소스 파일 삭제 |

### 3.2 산출물 생성 및 다운로드

| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/reports/weekly-sources/generate` | 산출물 생성 |
| GET | `/api/reports/weekly-outputs` | 산출물(job) 목록 조회 |
| GET | `/api/reports/weekly-outputs/download?job=&file=` | 산출물 파일 다운로드 |
| DELETE | `/api/reports/weekly-outputs/:jobId` | 산출물(job) 삭제 |

### 3.3 산출물 생성 API 요청/응답

**POST /api/reports/weekly-sources/generate**

**Request Body:**
```json
{
  "sourceId": "파일명.xlsx",        // 필수(소스 기반 생성 시)
  "useAutoPeriod": 1,               // 주간(자동) 시 1 전송
  "startDate": "2026-02-02",        // 기간 지정 시
  "endDate": "2026-02-08"           // 기간 지정 시
}
```

**Response (201 Created):**
```json
{
  "success": true,
  "data": {
    "jobId": "job_1739052000000",
    "outputDir": "data/weekly-outputs/job_xxx",
    "file": "PUBGMOBILE_모니터링_주간보고서_20260202_20260208.xlsx",
    "periodStart": "2026-02-02",
    "periodEnd": "2026-02-08",
    "message": "산출물이 생성되었습니다."
  }
}
```

**기간 적용 규칙:**
- `useAutoPeriod === 1` 또는 `startDate`/`endDate` 미전송 → **한국시간(Asia/Seoul) 달력 기준 직전 완료 주**의 **월요일~일요일** (VoC 파일 내 날짜와 무관). 구현: `backend/utils/kstWeek.util.js` 의 `getLastCompletedKstWeekMonSun`
- `startDate`, `endDate` 모두 전송 → 사용자 지정 기간 (직전 비교 구간은 기존 `getPreviousWeekRange` 로직)

## 5. VoC 파싱 및 기간 계산

### 5.1 컬럼 감지
- **헤더 기반 자동 감지**: 1~5행 스캔, 키워드 매칭으로 컬럼 인덱스 결정
- **키워드 매핑**:
  - 날짜: 날짜, date, 일자
  - 대분류: 대분류, category, 분류
  - 중분류: 중분류, sub_category
  - 성향: 성향, sentiment
  - 내용: 내용, content
  - 요약: 요약, 요약문, i_col, summary
  - 건수: 건수, 실제 건수, count, 수량, 합계
- **감지 실패 시**: 기본값 사용 (날짜 B열부터 시작: B=날짜, D=대분류, E=중분류, G=성향, H=내용, J=요약, X=건수)

### 5.2 날짜 파싱 지원 형식
- Excel serial 숫자
- `YYYY-MM-DD`, `YYYY.MM.DD`, `YYYY/MM/DD`
- `YYYYMMDD` (8자리)
- Date 객체, `new Date()` 파싱 가능 문자열

### 5.3 기간 모드 (주간 자동)
- **이번 주**: VoC 최신일 기준 직전 월요일~일요일
  - 예: 최신일 2026-02-09(월) → 2026-02-02(월) ~ 2026-02-08(일)
- **지난 주**: 이번 주 시작일 직전 7일
  - 예: 이번 주 02-02~02-08 → 지난주 01-26~02-01

### 5.4 기간 모드 (기간 지정)
- 사용자 지정 시작일~종료일을 "이번 주"로 사용
- 지난 주: (시작일 - 1일) 기준 직전 N일 (N = 이번 주 일수)

## 7. 오류 처리

| 상황 | HTTP | 사용자 메시지 |
|------|------|---------------|
| 소스 파일 없음 | 404 | 파일을 찾을 수 없습니다. |
| VoC 시트 없음 | 500 | VoC 시트가 없습니다. |
| 유효한 데이터 없음 | 500 | VoC 시트에 유효한 데이터가 없습니다. |
| 기간 형식 오류 | 500 | 기간 형식이 올바르지 않습니다. (YYYY-MM-DD) |
| 시작일 > 종료일 | 500 | 시작일은 종료일보다 클 수 없습니다. |

## 9. 주의해야 할 사항들

### 9.1 소스 엑셀 파일
- **시트명**: 반드시 `VoC` (대소문자 구분). 첫 번째 시트가 VoC여야 함.
- **열 구조**: 날짜가 B열부터 시작하는 레이아웃 지원. A열이 비어 있어도 컬럼 감지 동작.
- **헤더**: 1~5행 중 어딘가에 "날짜", "대분류" 키워드가 있어야 컬럼 자동 감지가 동작함.
- **날짜 컬럼**: 파싱 불가 행은 제외됨. 날짜 형식이 비표준이면 `YYYY-MM-DD` 등 지원 형식으로 통일 권장.
- **건수 컬럼**: `count <= 0`인 행은 제외됨. 건수는 **감지된 건수 열(기본 X열) 합계**만 사용(행 개수 아님).

### 9.2 기간 및 데이터 범위
- **주간(자동)**: VoC 최신일 기준 직전 월~일. 최신일 이후 데이터가 없으면 해당 주에 0건.
- **기간 지정**: 지정한 시작일~종료일 구간에 VoC 데이터가 없으면 이번주/지난주 모두 0건.
- VoC 데이터의 날짜 범위와 선택(또는 자동) 기간이 겹치지 않으면 산출물이 비어 보일 수 있음.

### 9.3 컬럼 감지
- 헤더 감지 실패 시 B=날짜, X=건수 등 기본 매핑 사용(날짜가 B열부터 시작하는 구조 기준). 실제 구조와 다르면 파싱 오류 가능.
- 엑셀 구조 변경 시 `detectVocColumns`의 `headerKeywords`에 새 키워드 추가 검토.

### 9.4 LLM 동향 요약 시트
- `OPENAI_API_KEY` 환경변수가 없으면 해당 시트는 생성되지 않음.
- API 호출 실패 시 해당 시트만 생략되고, 나머지 시트는 정상 생성.

### 9.5 파일 및 디렉터리
- `data/weekly-sources`, `data/weekly-outputs`는 서버 기동 시 자동 생성.
- multer 업로드 경로·설정이 `weekly-sources`와 일치해야 함.
- 동시 다중 생성 요청 시 `jobId`로 분리되어 충돌 없음. 다만 대용량 파일 동시 처리 시 메모리·CPU 부하 주의.

### 9.6 프론트엔드·백엔드 연동
- Vite 프록시: `/api`, `/uploads` → `http://127.0.0.1:8081`. 백엔드 미기동 시 API 호출 실패.
- 백엔드(포트 8081), 프론트엔드(포트 5174) 둘 다 실행 중이어야 전체 기능 동작.
- 프론트엔드 시작 경로: 프로젝트 루트 (`npm run dev`)
- 백엔드 시작 경로: `backend/` (`node server.js`)

### 9.7 파싱 실패 시 진단
- 파싱 결과가 0건이거나 건수가 의심스러우면 서버 로그의 `[VocParse] loadVocSheet` 확인.
- `skippedNoDate`, `skippedNoCount`, `dateSamples` 등으로 원인 파악 가능.
