## 1. 서버 재시작

### 1.1 권장 방법: safe-start.sh

```bash
# 프로젝트 루트에서
cd /home/young-dev/AIM
bash scripts/safe-start.sh
```

**동작 요약**
- 기존 `server.js` 및 워커 프로세스 종료
- 8080 포트 해제
- **단일** `node server.js` 인스턴스만 백그라운드 실행
- 워커는 server.js의 자식 프로세스로만 구동 (중복 프로세스 방지)

### 1.2 프로세스 중복 방지

| 항목 | 설명 |
|------|------|
| **flock** | 동시에 여러 `safe-start.sh` 실행 차단 (중복 서버 spawn 방지) |
| **단일 인스턴스** | server.js 1개 + 그 자식 워커들만 유지 |
| **종료 순서** | server.js(부모) → 포트 8080 → 워커(고아 프로세스) 순으로 정리 |

### 1.3 스크립트 처리 순서

1. **[0/4]** server.js 메인 프로세스 종료
2. **[1/4]** 8080 포트 점유 프로세스 종료
3. **[2/4]** 워커 프로세스 종료 (고아 프로세스 정리)
4. **[3/4]** 프로세스 종료 대기 및 최종 확인
5. **[4/4]** `node server.js` 백그라운드 시작

## 2. 주요 파일

## 4. 문제 발생 시

### 4.1 "다른 safe-start가 이미 실행 중입니다"
- 다른 터미널에서 safe-start가 실행 중
- `ps aux | grep safe-start`로 확인 후 완료 대기
- 비정상 종료 시: `rm /tmp/aim-safe-start.lock` 후 재시도

### 4.2 워커 중복 실행 (라인 알림 N배 전송)
- `node backend/scripts/check-work-notifications.js` 로 확인
- `safe-start.sh` 실행하여 정리 후 서버 재시작

### 4.3 포트 8080 사용 중
- `lsof -i:8080` 로 프로세스 확인
- `safe-start.sh`가 해당 프로세스를 종료한 뒤 서버 시작

- **Chromium launch 옵션**: `--disable-extensions`, `--disable-sync`, `--mute-audio`, `--no-first-run` 등 메모리 절감 플래그 적용 (naverCafe, naverCafeClan, naverCafeBackfill, screenshot, manualIngest)

## 7. 메모리 안정화 원칙 (시스템 구현 룰)

> **기본 방침**: 서버/워커는 메모리 안정화를 최우선으로 설계한다. 실행마다 새 창·새 프로세스를 띄우는 방식은 금지한다.

### 7.1 서버 재시작

| 룰 | 설명 |
|----|------|
| **단일 인스턴스** | 서버 재시작 시 `server.js` 1개만 구동. 여러 창·여러 포트로 인스턴스를 나누지 않는다. |
| **flock 사용** | safe-start 등 재시작 스크립트는 flock으로 동시 실행을 막아 중복 spawn을 방지한다. |
| **부모-자식 일원화** | 워커는 server.js의 자식 프로세스로만 실행한다. 별도 독립 프로세스로 여는 방식은 사용하지 않는다. |

### 7.2 워커 실행

| 룰 | 설명 |
|----|------|
| **실행마다 창 띄우기 금지** | 워커 실행마다 새 Chromium·새 브라우저를 띄우지 않는다. 공유 브라우저 또는 스캔 시에만 launch 후 즉시 close 패턴을 사용한다. |
| **브라우저 공유 또는 단기 사용** | (a) 여러 스캔에 하나의 브라우저 인스턴스를 재사용하거나, (b) 스캔 직전 launch → 스캔 완료 후 close로 단기 사용한다. 상시 다중 브라우저 유지는 최소화한다. |
| **page·리소스 정리 의무** | `page`, `context` 등은 try/finally에서 반드시 close/dispose 한다. early return·에러 발생 시에도 누수가 없도록 함수 스코프를 보장한다. |

### 7.3 메모리 사용

| 룰 | 설명 |
|----|------|
| **캐시 상한** | in-memory 캐시(Map, Set 등)는 최대 크기 제한을 둔다. LRU 등으로 eviction을 적용한다. |
| **Chromium 옵션** | launch 시 `--disable-extensions`, `--disable-dev-shm-usage`, `--mute-audio` 등 메모리 절감 플래그를 반드시 사용한다. |
| **대용량 버퍼 회피** | Excel 등 대용량 처리 시 `writeBuffer()` 대신 `writeFile()` 또는 스트리밍을 우선 검토한다. |

### 7.4 금지 사항

- ❌ 워커 실행·스캔마다 새 Chromium launch
- ❌ 서버 재시작 시 동시 다중 인스턴스 spawn
- ❌ 무제한 캐시(Map/Set 등) 누적
- ❌ page·browser를 close 없이 계속 유지
