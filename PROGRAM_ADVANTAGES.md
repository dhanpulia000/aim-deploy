## 1. 실시간 모니터링 및 이슈 관리

### 1.1 실시간 데이터 업데이트
- **WebSocket 기반 실시간 통신**: 페이지 새로고침 없이 즉시 데이터 업데이트
- **자동 새로고침**: 설정 가능한 간격으로 자동 데이터 갱신
- **실시간 상태 표시**: 에이전트 상태, 이슈 처리 상황 실시간 반영
- **즉각적인 알림**: 새로운 이슈 발생 시 즉시 화면에 표시

### 1.2 포괄적인 이슈 추적
- **다중 소스 통합**: Discord, 네이버 카페, Slack, 수동 입력 등 모든 소스 통합 관리
- **이슈 상세 정보**: 제목, 내용, 심각도, 분류, 담당자, 발생 시간 등 모든 정보 한눈에 확인
- **이슈 히스토리**: 상태 변경 이력, 댓글, 처리 과정 전체 추적
- **스크린샷 자동 첨부**: 이슈 발생 시 자동으로 스크린샷 캡처 및 저장

### 1.3 스마트 필터링 및 검색
- **다중 필터 조합**: 소스, 심각도, 상태, 담당자, 날짜 등 다양한 조건 동시 적용
- **실시간 필터링**: 필터 변경 시 즉시 결과 반영
- **프로젝트별 분리**: 여러 프로젝트를 독립적으로 관리
- **검색 기능**: 키워드 기반 빠른 검색
- **저장된 필터**: 자주 사용하는 필터 조합 저장 및 재사용

### 1.4 이슈 분류 및 우선순위 관리
- **자동 분류**: AI 기반 자동 카테고리 분류
- **수동 분류 지원**: 필요 시 수동으로 분류 변경 가능
- **심각도 레벨**: Sev1, Sev2, Sev3, Info 등 명확한 심각도 구분
- **중요도 표시**: HIGH, MEDIUM, LOW 중요도 시각적 표시
- **SLA 추적**: 각 이슈의 SLA 임박 여부 자동 계산 및 표시

### 1.5 이슈 처리 워크플로우
- **상태 관리**: OPEN, IN_PROGRESS, RESOLVED, CLOSED 등 명확한 상태 전환
- **담당자 할당**: 이슈별 담당자 지정 및 변경
- **처리 시간 추적**: 이슈 처리에 소요된 시간 자동 계산
- **완료 처리**: 간편한 완료 처리 및 상태 업데이트
- **이슈 잠금**: 동시 편집 방지를 위한 이슈 잠금 기능

## 2. AI 기반 지능형 기능

### 2.1 RAG 기반 AI 어시스턴트
- **컨텍스트 기반 답변**: 업무 가이드 문서를 기반으로 정확한 답변 제공
- **벡터 검색**: 의미 기반 유사도 검색으로 관련 문서 자동 찾기
- **자연어 질의**: 일반적인 질문으로 업무 가이드 검색
- **관련 문서 제시**: 질문과 관련된 여러 가이드 문서 동시 제공
- **대화형 인터페이스**: 채팅 형태의 직관적인 UI

### 2.2 AI 자동 분류
- **자동 카테고리 분류**: 이슈 내용 분석하여 적절한 카테고리 자동 할당
- **학습 가능**: 분류 규칙 학습 및 개선
- **정확도 향상**: 지속적인 학습으로 분류 정확도 향상
- **수동 보정 지원**: AI 분류 결과 수정 및 학습 반영

### 2.3 업무 가이드 관리
- **PDF 파일 지원**: PDF 파일 업로드 및 자동 파싱
- **임베딩 자동 생성**: 업로드된 문서 자동 임베딩 처리
- **한글 파일명 지원**: 한글 파일명 정상 처리
- **일괄 업로드**: 여러 파일 동시 업로드 및 처리
- **가이드 검색**: 제목, 내용, 태그 기반 검색
- **카테고리별 분류**: 가이드를 카테고리별로 체계적 관리
- **우선순위 설정**: 가이드별 우선순위 설정으로 검색 결과 정렬

### 2.4 지능형 추천
- **유사 이슈 추천**: 과거 유사한 이슈 처리 방법 제시
- **담당자 추천**: 이슈 유형에 따른 적절한 담당자 추천
- **처리 방법 제안**: AI가 분석한 최적 처리 방법 제안

## 3. 다채널 통합 및 자동화

### 3.1 Discord 통합
- **실시간 모니터링**: Discord 채널 메시지 실시간 수집
- **자동 이슈 생성**: 특정 조건의 메시지 자동으로 이슈 변환
- **댓글 수집**: 게시글의 댓글 자동 수집 및 분석
- **중복 방지**: 동일 메시지 중복 수집 방지

### 3.2 네이버 카페 통합
- **다중 카페 지원**: 여러 네이버 카페 동시 모니터링
- **게시판별 필터링**: 특정 게시판만 선택적 모니터링
- **키워드 필터링**: 특정 키워드 포함 게시글만 수집
- **백필 작업**: 과거 게시글 일괄 수집
- **클랜 게시판 지원**: 클랜 게시판 전용 모니터링
- **댓글 수 집계**: 게시글별 댓글 수 자동 집계

### 3.3 Slack 통합
- **공지사항 자동 수집**: Slack 채널의 공지사항 자동 수집
- **이슈 공유 기능**: 이슈를 Slack 채널로 공유
- **이미지 업로드**: 스크린샷 이미지 자동 업로드
- **Block Kit 포맷**: Slack의 Block Kit 형식으로 보기 좋은 메시지 전송
- **채널 선택**: 고객사 채널/내부 채널 선택적 공유
- **커스텀 메시지**: 공유 시 추가 메시지 작성 가능

### 3.4 LINE 알림 시스템
- **업무 알림 전송**: 지정된 시간에 LINE 메시지 자동 전송
- **다양한 반복 설정**: 매일, 매주, 매월, 특정 날짜, 간격별 반복
- **그룹방 지원**: LINE 그룹방에 알림 전송
- **알림 복사 기능**: 기존 알림 복사하여 시간만 변경 가능
- **반복 등록**: 하루에 여러 번 동일 알림 등록 가능
- **중복 전송 방지**: 동일 알림 중복 전송 자동 방지
- **자동 답장 비활성화**: 그룹방에서 봇 자동 답장 방지

### 3.5 자동화 워커
- **백그라운드 처리**: 백그라운드에서 지속적 작업 수행
- **안정적인 실행**: 프로세스 크래시 시 자동 재시작
- **에러 핸들링**: 에러 발생 시 로깅 및 복구
- **리소스 관리**: 메모리 및 CPU 사용량 최적화
- **로그 기록**: 모든 작업 상세 로그 기록

## 4. 사용자 경험 (UX)

### 4.1 직관적인 인터페이스
- **깔끔한 디자인**: 모던하고 깔끔한 UI 디자인
- **다크 모드 지원**: 눈의 피로를 줄이는 다크 모드
- **반응형 디자인**: 데스크톱, 태블릿, 모바일 모든 기기 지원
- **직관적인 네비게이션**: 메뉴 구조가 명확하고 사용하기 쉬움
- **시각적 피드백**: 버튼 클릭, 로딩 상태 등 명확한 시각적 피드백

### 4.2 빠른 성능
- **가상 스크롤**: 대량의 데이터도 부드럽게 스크롤
- **지연 로딩**: 필요한 데이터만 점진적으로 로드
- **캐싱**: 자주 사용하는 데이터 캐싱으로 빠른 응답
- **최적화된 렌더링**: React 최적화 기법으로 빠른 렌더링
- **이미지 최적화**: 이미지 로딩 최적화 및 에러 처리

### 4.3 편의 기능
- **드래그 앤 드롭**: 파일 업로드 시 드래그 앤 드롭 지원
- **키보드 단축키**: 자주 사용하는 기능의 키보드 단축키
- **자동 저장**: 입력 중인 내용 자동 저장
- **복사 기능**: 이슈 정보, 링크 등 원클릭 복사
- **새 탭에서 열기**: 외부 링크 새 탭에서 열기
- **이미지 확대**: 스크린샷 클릭 시 전체 화면으로 확대

### 4.4 사용자 맞춤 설정
- **개인 설정**: 사용자별 화면 설정 저장
- **필터 저장**: 자주 사용하는 필터 조합 저장
- **테마 설정**: 다크/라이트 모드 선택
- **알림 설정**: 개인별 알림 설정

### 4.5 접근성
- **키보드 네비게이션**: 마우스 없이도 모든 기능 사용 가능
- **스크린 리더 지원**: 시각 장애인을 위한 스크린 리더 지원
- **명확한 라벨**: 모든 버튼과 입력 필드에 명확한 라벨
- **색상 대비**: 충분한 색상 대비로 가독성 향상

## 5. 관리 및 운영 기능

### 5.1 대시보드 및 KPI
- **실시간 KPI**: 열린 이슈 수, Sev1 이슈, SLA 임박 이슈 등 핵심 지표
- **에이전트 상태**: 각 에이전트의 현재 상태, 처리 중인 티켓 수
- **오늘 처리량**: 오늘 처리한 이슈 수 실시간 표시
- **시각적 차트**: Recharts를 활용한 직관적인 데이터 시각화
- **트렌드 분석**: 시간대별 이슈 발생 추이 분석

### 5.2 에이전트 관리
- **에이전트 등록**: 새로운 에이전트 추가 및 정보 관리
- **근무 타입 설정**: 다양한 근무 타입 지원 (오전, 오후, 정오, 야간 등)
- **스케줄 관리**: 에이전트별 근무 스케줄 관리
- **성과 분석**: 에이전트별 처리량, 평균 처리 시간 등 성과 분석
- **상태 모니터링**: 에이전트의 현재 상태 실시간 모니터링

### 5.3 프로젝트 관리
- **다중 프로젝트**: 여러 프로젝트 동시 관리
- **프로젝트별 분리**: 프로젝트별로 데이터 완전 분리
- **프로젝트 설정**: 프로젝트별 개별 설정
- **프로젝트 전환**: 간편한 프로젝트 전환

### 5.4 카테고리 관리
- **계층적 구조**: 카테고리 그룹 > 카테고리 2단계 구조
- **유연한 분류**: 다양한 이슈 유형에 대응하는 유연한 분류 체계
- **자동 매칭**: AI 기반 자동 카테고리 매칭
- **수동 조정**: 필요 시 수동으로 카테고리 조정

### 5.5 공지사항 관리
- **공지사항 수집**: Slack, 네이버 카페 등에서 공지사항 자동 수집
- **공지사항 페이지**: 모든 공지사항을 한 곳에서 관리
- **필터링**: 날짜, 주차, 담당자, 게임별 필터링
- **열람 확인**: 에이전트별 공지사항 열람 상태 추적
- **확인 버튼**: 에이전트가 직접 열람 확인 표시

### 5.6 리포트 생성
- **주간 리포트**: 주간 이슈 처리 리포트 자동 생성
- **Excel 내보내기**: 리포트를 Excel 파일로 내보내기
- **Google Sheets 연동**: Google Sheets로 자동 업로드
- **커스텀 리포트**: 필요한 항목만 선택하여 리포트 생성
- **일일 리포트**: 일일 처리 현황 리포트

### 5.7 모니터링 제어
- **모니터링 시작/중지**: 각 소스별 모니터링 제어
- **실시간 상태**: 모니터링 워커의 실시간 상태 확인
- **로그 확인**: 모니터링 워커의 상세 로그 확인
- **설정 변경**: 모니터링 주기, 대상 등 설정 변경

## 6. 성능 및 기술적 장점

- **TypeScript**: 타입 안정성으로 버그 사전 방지
- **React 18**: 최신 React 기능 활용
- **Vite**: 빠른 빌드 및 개발 서버
- **Tailwind CSS**: 유틸리티 기반 빠른 스타일링
- **Node.js**: 비동기 처리로 높은 성능

- **SQLite**: 경량 데이터베이스로 빠른 성능
- **인덱싱**: 자주 조회하는 필드 인덱싱
- **쿼리 최적화**: 효율적인 쿼리로 빠른 응답
- **트랜잭션**: 데이터 일관성 보장

### 6.3 벡터 검색
- **pgvector 지원**: PostgreSQL + pgvector를 통한 벡터 검색
- **임베딩 저장**: 문서 임베딩 효율적 저장
- **유사도 검색**: 의미 기반 유사도 검색
- **빠른 검색**: 인덱싱된 벡터로 빠른 검색

### 6.4 확장 가능한 아키텍처
- **모듈화**: 기능별 모듈 분리로 유지보수 용이
- **서비스 레이어**: 비즈니스 로직과 데이터 접근 분리
- **컨트롤러 패턴**: API 엔드포인트 체계적 관리
- **워커 패턴**: 백그라운드 작업 독립적 실행

### 6.5 API 설계
- **RESTful API**: 표준 REST API 설계

- **자동 복구 시스템**: 워커 프로세스 크래시 시 자동 재시작 (5초 지연 후)
  - 프로세스 종료 코드 모니터링
  - 에러로 종료된 경우에만 자동 재시작
  - 정상 종료 시 재시작하지 않음
  - 재시작 타임아웃 관리로 중복 재시작 방지

  - 모든 서비스 레이어에 try-catch 블록 적용
  - safeQuery 함수로 데이터베이스 쿼리 안전 실행
  - 에러 발생 시 기본값 반환으로 시스템 중단 방지
  - 개별 작업 실패가 전체 시스템에 영향 주지 않음
- **로깅 시스템**: 
  - Winston 기반 구조화된 로깅
  - 일일 로그 로테이션 (14일 보관)
  - 에러 로그 별도 파일 저장
  - 로그 압축 및 자동 정리
  - 상세한 에러 스택 트레이스 기록
- **프로세스 관리**:
  - 좀비 프로세스 방지
  - 타임아웃 기반 강제 종료 (10초)
  - 프로세스 상태 실시간 모니터링
  - 안전한 종료 프로세스
- **데이터 백업**: Git을 통한 코드 백업 자동화
