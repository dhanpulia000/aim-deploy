# 댓글 수집 봇 차단 위험 분석

## 현재 구현 방식

### 1. 댓글 수(commentCount) 추출
- **위치**: 리스트 페이지에서 추출
- **방법**: DOM에서 댓글 수 텍스트 파싱
- **추가 요청**: 없음 (리스트 페이지 스캔 시 이미 수집)
- **봇 차단 위험**: ⭐ 매우 낮음 (리스트 페이지는 이미 방문해야 함)

### 2. 댓글 내용(scrapedComments) 수집
- **위치**: 상세 페이지에서 추출
- **방법**: 상세 페이지 방문 후 댓글 영역 DOM 파싱
- **추가 요청**: 핫토픽인 경우에만 1회 추가 방문
- **봇 차단 위험**: ⚠️ 임계값에 따라 다름

## 핫토픽 판단 기준

```javascript
const HOT_TOPIC_THRESHOLD = 5; // 현재 설정

isHotTopic = isWatchedAuthor || (commentCount >= 5);
shouldScrapeComments = isHotTopic; // 핫토픽인 경우에만 댓글 내용 수집
```

## 위험 분석

### 시나리오 1: 임계값 5개 (현재 설정)

**예상되는 비율** (일반적인 네이버 카페 게시글 분포):
- 댓글 0개: ~70-80%
- 댓글 1-4개: ~15-20%
- 댓글 5-9개: ~5-8% ⚠️
- 댓글 10개 이상: ~2-5%

**100개 게시글 스캔 시**:
- 리스트 페이지 방문: 1회 (필수)
- 상세 페이지 방문: 약 7-13개 (5개 이상 댓글인 경우)
- 총 추가 요청: 약 7-13회

**위험도**: 🟡 중간
- 한 번의 스캔에서는 위험하지 않음
- 하지만 자주 스캔하면 누적 요청이 많아질 수 있음

### 시나리오 2: 임계값 10개 (더 보수적인 설정)

**100개 게시글 스캔 시**:
- 리스트 페이지 방문: 1회 (필수)
- 상세 페이지 방문: 약 2-5개 (10개 이상 댓글인 경우)
- 총 추가 요청: 약 2-5회

**위험도**: 🟢 낮음
- 매우 안전한 수준
- 차단될 가능성 거의 없음

## 현재 보호 메커니즘

1. **랜덤 대기 시간**: 2분 30초 ~ 4분 사이 랜덤 대기
2. **재시도 로직**: 실패 시 재시도 (최대 3회)
3. **실제 브라우저 사용**: Playwright로 실제 브라우저 동작 시뮬레이션
4. **User-Agent 설정**: 최신 브라우저 User-Agent 사용
5. **쿠키 지원**: 로그인 상태 유지 (있는 경우)

## 권장 사항

### 옵션 1: 임계값 유지 (5개) - 현재 설정
- ✅ 더 많은 핫토픽 감지 가능
- ⚠️ 상세 페이지 방문이 많아질 수 있음
- 📌 모니터링 필요: 실제 스캔 시 상세 페이지 방문 횟수 확인

### 옵션 2: 임계값 상향 (10개) - 더 안전
- ✅ 차단 위험 최소화
- ✅ 대부분의 중요한 게시글(댓글 많은 글)은 여전히 감지
- ⚠️ 5-9개 댓글 게시글은 댓글 내용 미수집

### 옵션 3: 하이브리드 접근
- 환경 변수로 임계값 조정 가능 (현재 지원됨)
- 운영 환경에서는 10개, 테스트 환경에서는 5개 사용

## 결론

**현재 설정(5개)은 합리적이지만, 다음을 권장합니다:**

1. **환경 변수 활용**: `NAVER_CAFE_HOT_TOPIC_THRESHOLD=10` 설정 고려
2. **모니터링**: 실제 스캔 시 상세 페이지 방문 횟수 로깅 및 확인
3. **점진적 적용**: 먼저 높은 임계값으로 시작 후 필요시 조정

**특히 중요한 것**:
- 댓글 **수**는 리스트 페이지에서 이미 추출되므로 추가 위험 없음
- 댓글 **내용**만 상세 페이지 방문이 필요하므로, 임계값 조정으로 제어 가능
