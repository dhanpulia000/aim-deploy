# 크롤러 본문 추출 로직 분석

## 현재 구현된 로직

### 1. 본문 추출 셀렉터 (우선순위 순)
```javascript
const contentSelectors = [
  '#articleBodyContents',
  '.article_view .se-main-container',
  '.article_view .se-component',
  '.ArticleContent',
  '.se-main-container',
  '#content-area',
  '.article_view',
  '.se-viewer',
  '.se-section-text'
];
```

### 2. 추출 과정
1. 각 셀렉터를 순서대로 시도
2. 요소를 찾으면 `textContent` 추출
3. 제목으로 시작하면 제목 부분 제거
4. ": 네이버 카페" 텍스트 제거
5. 최소 10자 이상이어야 본문으로 인정
6. 조건을 만족하면 해당 텍스트를 본문으로 사용

## 발견된 문제점

### 문제 1: 셀렉터가 실제 DOM 구조와 맞지 않음
- Naver Cafe는 동적으로 콘텐츠를 로드할 수 있음
- 페이지 구조가 변경되었을 수 있음
- 셀렉터가 너무 구체적이어서 실제 구조와 맞지 않을 수 있음

### 문제 2: 제목 제거 로직의 한계
- `text.startsWith(cleanTitle)` 조건이 정확하지 않을 수 있음
- 제목이 본문 중간에 포함될 수 있음
- 제목과 본문이 완전히 동일한 경우 처리 불가

### 문제 3: 최소 길이 조건이 너무 엄격함
- `text.length > 10` 조건으로 인해 짧은 본문이 무시될 수 있음
- 실제로 본문이 없는 게시글과 구분이 어려움

### 문제 4: 비동기 콘텐츠 로딩
- `waitUntil: 'networkidle'`로는 부족할 수 있음
- JavaScript로 동적 로드되는 콘텐츠를 기다리지 못할 수 있음

### 문제 5: 로깅 부족
- 어떤 셀렉터가 성공했는지, 실패했는지 알 수 없음
- 디버깅이 어려움

## 개선 방안

### 1. 더 많은 셀렉터 추가
- Naver Cafe의 다양한 페이지 구조 대응
- 더 일반적인 셀렉터 사용

### 2. 대기 시간 추가
- 특정 요소가 나타날 때까지 대기
- `page.waitForSelector()` 사용

### 3. 제목 제거 로직 개선
- 정규식으로 더 정확하게 제목 제거
- 여러 패턴 시도

### 4. 디버깅 로깅 추가
- 각 셀렉터 시도 결과 로깅
- 추출된 텍스트 길이 로깅

### 5. 대체 방법 시도
- 셀렉터 실패 시 다른 방법 시도
- 전체 페이지 텍스트에서 제목 제거 후 본문 추출
