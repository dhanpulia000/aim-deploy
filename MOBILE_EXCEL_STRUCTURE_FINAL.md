# Mobile 모니터링 일일 보고서 Excel 파일 구조 (최종)

## 전체 시트 구성

### 1. Summary 시트
**용도**: 통계 요약
- 종합: 총게시물량, 댓글량, 취합량
- 주요 분류별: 불법 프로그램, 최적화, 버그
- 대분류별: 의견/건의/문의/제보 건수

### 2. Data 시트 (SUMMARY DATA)
**용도**: 주간 데이터 요약
- 주차 (12월 5주차, 1월 1주차 등)
- 날짜 (YYYY-MM-DD)
- 작성자
- 커뮤니티 이슈

### 3. Issue 시트
**용도**: 이슈 상세 정보
- 날짜, 분류, 요약, 세부 내용
- 테스트 정보 (재현 여부, 테스트 기기, OS 버전)
- 이슈 확인 시간

### 4. Cheat&Abuse 시트
**용도**: 비인가 프로그램 제보 내역
- 날짜, 신고자 정보 (닉네임, 네이버ID)
- 신고 대상자 (Ingame nickname, CharacterID)
- 제보 채널, Description (AIMBOT, ESP, 지형 투과 등)
- URI, 제보 게시판, 양식 준수 여부
- 커뮤니티 제재 이력, 증거 자료

### 5. VOC 시트 (이용자 동향)
**용도**: 사용자 피드백 목록
- 출처, 대분류, 중분류
- 종류 (문의/의견/건의)
- 성향 (부정/중립/긍정)
- 중요도 (중/하)

## 데이터 변환 규칙

### Data 시트
- 주차 정보 유지: "12월 5주차", "1월 1주차" 등
- 날짜 파싱: YYYY-MM-DD 형식
- 상태: 기본값 'resolved' (이미 처리된 데이터)
- 카테고리: '커뮤니티'
- 심각도: 3 (기본값)

### Cheat&Abuse 시트
- 불법 프로그램 타입: Description에서 AIMBOT, ESP, 지형 투과, 무고벤 추출
- 심각도: AIMBOT/ESP → 1, 무고벤 → 2
- 상태: 기본값 'triage'
- Source: 'naver'
- Sentiment: 'neg'

### Issue 시트
- 심각도: 세부 내용 키워드 분석 (비정상 종료/오류 → 2)
- 시간: 날짜 + 이슈확인시간 결합 → `createdAt`

### VOC 시트
- 심각도: 중요도 (중→2, 하→3) 또는 종류 기반 자동 분류
- 출처: Naver 포함 → 'naver', 기타 → 'system'
- 성향: 부정→neg, 중립→neu, 긍정→pos

## 사용 방법

1. **Excel 파일 업로드** → `/login` → `/dashboard`
2. **파일 타입 선택** → "Mobile 모니터링 일일 보고서"
3. **자동 파싱** → Issue, VOC 시트 자동 변환
4. **주간 보고서 생성** → 일일 데이터 기반 자동 생성
