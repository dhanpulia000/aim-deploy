## 🔍 파일 구조 확인 방법

터미널에서 다음 명령어로 각 파일의 구조를 확인할 수 있습니다:

```bash
cd backend
node check-excel.js
```

이 스크립트는:
- 각 파일의 시트 개수와 이름
- 각 시트의 행/열 수
- 헤더 정보
- 샘플 데이터

를 출력합니다.

---

## 💡 파일 처리 방법

### 현재 시스템에서 처리되는 파일

✅ **wallboard-data.xlsx**
- Agents 시트 → 에이전트 정보
- Tickets 시트 → 티켓/이슈 정보
- 자동 읽기 및 업데이트

### 향후 처리 가능한 파일

다음 파일들도 업로드 기능으로 처리 가능합니다:

1. **일일 보고서** (PC/Mobile)
   - 업로드 시 자동 파싱
   - 데이터 추출 및 저장

2. **주간 보고서** (PC/Mobile)
   - 주간 종합 데이터 파싱
   - 통계 정보 추출

3. **이슈 정리**
   - 이슈 목록 파싱
   - 티켓 데이터로 변환 가능

---

## 📊 데이터 연동 예정

각 파일의 실제 구조를 확인한 후:
1. 파싱 로직 구현
2. 현황판 데이터로 변환
3. 자동 업데이트 기능 추가

가능합니다.
